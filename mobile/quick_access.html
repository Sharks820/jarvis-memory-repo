How Jarvis Works

Desktop Engine runs on your PC as a daemon. It handles all intelligence: memory, knowledge graph, LLM routing, proactive analysis, and automation.

Android App acts as your mobile interface: voice, notifications, calls, location, camera. It sends sensor data to the desktop brain and displays responses.

Intelligence Routing: Commands are classified by intent, then routed to the best LLM provider (Anthropic, Groq, Mistral, Z.ai, or local Ollama) based on complexity and privacy needs. Privacy-sensitive queries always stay local.

Memory: Everything Jarvis learns is stored in a local SQLite database with full-text search and vector embeddings. Knowledge is organized into branches (health, work, gaming, family, etc.) with confidence scores and contradiction detection.

Anti-Regression: Golden tasks are periodically re-evaluated to ensure intelligence scores never decline. Memory health is monitored for duplicates and staleness.

Security: All API calls use HMAC-SHA256 signing with nonce replay protection. Sync payloads are Fernet-encrypted. Master password required for sensitive operations.
