Create the two core C# classes that form the bridge server.

1. **Editor/ReflectionCommandDispatcher.cs** — namespace Jarvis.EditorBridge:
   - `Dictionary<string, List<MethodInfo>> _cache` — keyed by "TypeName.MethodName", stores ALL overloads per key (not first-wins, per Pitfall 3 in research)
   - `BuildCache()` — scan typeof(UnityEditor.EditorApplication).Assembly and typeof(UnityEngine.GameObject).Assembly; iterate all types, public static methods; add to cache
   - `Dispatch(string methodKey, JObject args)`:
     - Lookup methodKey in cache; throw KeyNotFoundException if missing
     - If multiple overloads: filter by matching parameter count (args.Count)
     - Call TypeCoercer.Coerce(method.GetParameters(), args) to convert JSON params
     - Call method.Invoke(null, parameters)
     - Return result (may be null for void methods)
   - Log cache size at build time: `Debug.Log($"[Jarvis] Reflection cache built: {totalMethods} methods from {totalTypes} types")`

2. **Editor/JarvisEditorBridge.cs** — namespace Jarvis.EditorBridge:
   - `[InitializeOnLoad]` attribute on static class
   - Private static fields: WebSocketServer _server, ReflectionCommandDispatcher _dispatcher
   - Static constructor:
     - Call StopServer() first (cleanup from previous domain)
     - Create _dispatcher, call BuildCache()
     - Call StartServer()
     - Register callbacks:
       - `EditorApplication.quitting += StopServer`
       - `AssemblyReloadEvents.beforeAssemblyReload += StopServer`
       - `AssemblyReloadEvents.afterAssemblyReload += BroadcastReady`
   - `StartServer()`:
     - `_server = new WebSocketServer("ws://localhost:8091")`
     - `_server.ReuseAddress = true` (prevents port binding failure on reload, per Pitfall 2)
     - `_server.AddWebSocketService<JarvisBridgeService>("/jarvis")`
     - Wrap `_server.Start()` in try/catch; on SocketException, retry once via `EditorApplication.delayCall` after 500ms
     - `Debug.Log("[Jarvis] Editor Bridge started on ws://localhost:8091/jarvis")`
   - `StopServer()`:
     - Guard: if `_server != null && _server.IsListening` then `_server.Stop()`
     - Set `_server = null`
   - `BroadcastReady()`:
     - Send `{"status":"ready"}` JSON to all connected WebSocket sessions
     - Access sessions via `_server.WebSocketServices["/jarvis"].Sessions`
   - `SendNotification(string method, object data)`:
     - Build JSON-RPC notification (no id): `{"jsonrpc":"2.0","method":method,"params":data}`
     - Broadcast to all sessions
   - Inner class `JarvisBridgeService : WebSocketBehavior`:
     - `OnOpen()`: Send `{"status":"ready"}` immediately to this session
     - `OnMessage(MessageEventArgs e)`:
       - Deserialize e.Data to JsonRpcRequest
       - Validate method is not empty
       - If method involves file write: call StaticAnalysisGuard (defense-in-depth)
       - Try _dispatcher.Dispatch(request.Method, request.Params)
       - On success: Send JsonRpcResponse.Success(request.Id, result)
       - On exception: Send JsonRpcResponse.Failure(request.Id, -32603, ex.Message)
     - `OnError(ErrorEventArgs e)`: Log via Debug.LogError
     - `OnClose(CloseEventArgs e)`: Log via Debug.Log

**Domain reload sequence:** When a .cs file is written to the project:
  1. Unity detects file change, begins compilation
  2. `beforeAssemblyReload` fires -> StopServer() gracefully closes WebSocket
  3. AppDomain is torn down (all static state destroyed)
  4. New AppDomain created, [InitializeOnLoad] fires static constructor again
  5. BuildCache() rebuilds reflection cache
  6. StartServer() binds port 8091 again (ReuseAddress=true handles lingering socket)
  7. `afterAssemblyReload` fires -> BroadcastReady() sends {"status":"ready"}
  8. Python client receives heartbeat, exits WAITING_FOR_BRIDGE state

**Path jail enforcement in bridge (defense-in-depth):** For any Dispatch call where the method involves file operations (detected by method name containing "Write", "Create", "Delete", "Move", "Copy" on file-related types), validate that path parameters start with Application.dataPath + "/JarvisGenerated" after Path.GetFullPath normalization. Reject with UnauthorizedAccessException if outside jail.
