    **ContextDetector** (`feature/context/ContextDetector.kt`):
    Hilt `@Singleton` injecting `@ApplicationContext context: Context`, `JarvisApiClient`.

    ```kotlin
    enum class UserContext(val label: String) {
        NORMAL("Normal"),
        MEETING("In Meeting"),
        DRIVING("Driving"),
        SLEEPING("Sleeping"),
        GAMING("Gaming"),
    }

    data class ContextState(
        val context: UserContext,
        val confidence: Float,
        val detectedAt: Long = System.currentTimeMillis(),
        val source: String,  // "calendar", "accelerometer", "time", "gaming_sync", "manual"
    )
    ```

    Methods:
    - `suspend fun detectCurrentContext(): ContextState` -- Runs all detectors in priority order (highest priority wins):
      1. **Gaming check** (priority 1): Query desktop `/settings` endpoint for `gaming_mode.enabled`. If true, return GAMING with source "gaming_sync".
      2. **Calendar meeting check** (priority 2): Use `ContentResolver` to query `CalendarContract.Events` for events happening right now (DTSTART <= now <= DTEND). If found, return MEETING with source "calendar", confidence based on how far into the meeting (early = higher confidence).
      3. **Driving check** (priority 3): Use Android `SensorManager` to read accelerometer data. Detect driving pattern: sustained vibration pattern (variance in acceleration > threshold over 30-second window). This is a simplified heuristic:
         - Register `TYPE_ACCELEROMETER` listener
         - Collect samples for 10 seconds
         - Calculate variance of magnitude (`sqrt(x^2 + y^2 + z^2)`)
         - If variance is in driving range (0.5 - 5.0 m/s^2 -- sustained vibration but not walking), and total magnitude bias suggests vehicle movement, return DRIVING.
         - NOTE: This is a heuristic approximation. More accurate would use Activity Recognition API, but that requires Google Play Services dependency. Keep it simple with accelerometer for now.
      4. **Sleep check** (priority 4): Check current time against configured sleep window (default 11 PM - 7 AM). If current time is in window and device is stationary (low accelerometer variance < 0.2), return SLEEPING with source "time".
      5. **Default**: Return NORMAL.

    - `fun registerAccelerometerListener()` / `fun unregisterAccelerometerListener()` -- Manage sensor registration lifecycle.

    IMPORTANT: Accelerometer access does not need special permissions. The SensorManager is available to any app. But keep sensor registration scoped to avoid battery drain -- only register when the service is actively checking context, not permanently.

    ```kotlin
    @Inject lateinit var contextDetector: ContextDetector
    @Inject lateinit var contextAdjuster: ContextAdjuster
    @Inject lateinit var contextStateDao: ContextStateDao

    private var lastContextCheckMs = 0L
    private var currentContext: UserContext = UserContext.NORMAL

    // In sync loop:
    val now = System.currentTimeMillis()
    if (now - lastContextCheckMs > 120_000L) { // every 2 minutes
        lastContextCheckMs = now
        try {
            val state = contextDetector.detectCurrentContext()
            if (state.context != currentContext) {
                currentContext = state.context
                contextAdjuster.applyContext(state)
                contextStateDao.insert(ContextStateEntity(
                    context = state.context.name,
                    confidence = state.confidence,
                    source = state.source,
                ))
                Log.i(TAG, "Context changed to: ${state.context.label}")
            }
        } catch (e: Exception) {
            Log.w(TAG, "Context detection error: ${e.message}")
        }
    }
    ```
