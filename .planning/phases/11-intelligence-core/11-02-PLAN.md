    **SchedulingCueExtractor** (`feature/scheduling/SchedulingCueExtractor.kt`): Hilt `@Singleton` class with no dependencies. Provides:

    ```kotlin
    data class SchedulingCue(
        val title: String,
        val dateTime: Long,          // epoch millis, 0 if only date (no time)
        val endDateTime: Long,       // epoch millis, 0 if not specified (default 1hr after start)
        val location: String,
        val sourcePackage: String,
        val sourceText: String,
        val confidence: Float,       // 0.0-1.0 based on how many cues matched
    )

    fun extract(text: String, sourcePackage: String): List<SchedulingCue>
    ```

    Extraction logic using regex patterns:
    - **Date patterns**: Match "January 15", "Jan 15", "1/15/2026", "2026-01-15", "tomorrow", "next Monday", "next week", "this Friday". Use `java.time.LocalDate` parsing.
    - **Time patterns**: Match "3:00 PM", "3pm", "15:00", "at 3", "at noon", "at midnight". Use `java.time.LocalTime` parsing.
    - **Location patterns**: Match "at [Location]" (capitalized words after "at" that are NOT time-related), "@ [Location]", addresses with street numbers (e.g., "123 Main St").
    - **Title inference**: Use the first line of the notification or the text before the date/time as the event title. Truncate to 100 chars.
    - **Confidence scoring**: 0.3 for date-only, 0.5 for date+time, 0.7 for date+time+location, 0.9 for date+time+location+title-like text.
    - Return empty list if no date pattern matches. Each distinct date found creates a separate SchedulingCue.

    IMPORTANT: Handle relative dates ("tomorrow", "next Monday") by computing against `LocalDate.now()` with `java.time.DayOfWeek`. Handle "next week" as next Monday.

    **JarvisNotificationListenerService** (`feature/scheduling/JarvisNotificationListenerService.kt`):
    Extend `android.service.notification.NotificationListenerService`. This service receives all notifications system-wide when enabled by the user in Android Settings > Notification Access.

    ```kotlin
    @AndroidEntryPoint
    class JarvisNotificationListenerService : NotificationListenerService() {
        @Inject lateinit var cueExtractor: SchedulingCueExtractor
        @Inject lateinit var calendarCreator: CalendarEventCreator

        // Package filter: only process notifications from SMS and email apps
        private val SCHEDULING_PACKAGES = setOf(
            "com.google.android.apps.messaging",  // Google Messages
            "com.samsung.android.messaging",       // Samsung Messages
            "com.google.android.gm",               // Gmail
            "com.microsoft.office.outlook",         // Outlook
            "com.samsung.android.email.provider",   // Samsung Email
        )

        private val scope = CoroutineScope(SupervisorJob() + Dispatchers.IO)

        override fun onNotificationPosted(sbn: StatusBarNotification) {
            val pkg = sbn.packageName
            if (pkg !in SCHEDULING_PACKAGES) return

            // Check if scheduling extraction is enabled
            val prefs = getSharedPreferences("jarvis_prefs", Context.MODE_PRIVATE)
            if (!prefs.getBoolean("scheduling_extraction_enabled", true)) return

            val extras = sbn.notification.extras
            val title = extras.getCharSequence(Notification.EXTRA_TITLE)?.toString() ?: ""
            val text = extras.getCharSequence(Notification.EXTRA_TEXT)?.toString() ?: ""
            val bigText = extras.getCharSequence(Notification.EXTRA_BIG_TEXT)?.toString() ?: ""

            // Use bigText if available (more content), fall back to text
            val fullText = "$title\n${bigText.ifBlank { text }}"
            if (fullText.isBlank() || fullText.length < 10) return

            scope.launch {
                try {
                    val cues = cueExtractor.extract(fullText, pkg)
                    for (cue in cues) {
                        if (cue.confidence < 0.3f) continue  // skip very low confidence

                        // Auto-create calendar event if confidence >= threshold
                        val autoCreateThreshold = prefs.getFloat("scheduling_auto_create_threshold", 0.5f)
                        if (cue.confidence >= autoCreateThreshold) {
                            val eventId = calendarCreator.createEvent(cue)
                            if (eventId > 0) {
                                calendarCreator.notifyDesktopOfEvent(cue)
                            }
                        }
                    }
                } catch (e: Exception) {
                    Log.w("JarvisScheduling", "Cue extraction failed: ${e.message}")
                }
            }
        }

        override fun onNotificationRemoved(sbn: StatusBarNotification) {
            // No action needed on removal
        }

        override fun onDestroy() {
            scope.cancel()
            super.onDestroy()
        }
    }
    ```
