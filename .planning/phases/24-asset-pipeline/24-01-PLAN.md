    Create BlenderTool class:

    1. `__init__(self, blender_path: str | None = None)` -- discovers Blender path:
       - First checks blender_path arg
       - Then os.environ.get("BLENDER_PATH")
       - Then default: "C:/Program Files/Blender Foundation/Blender 4.3/blender.exe"
       - Stores path but does NOT validate existence at init time (lazy -- validated at execute time)
    2. `async execute(self, *, script: str, input_path: str = "", output_path: str = "", **extra_args: str) -> dict[str, str]`:
       - Resolves script name to path in blender_scripts/ directory (e.g., "optimize_mesh" -> blender_scripts/optimize_mesh.py)
       - Builds arg list: blender_path, "--background", "--python", script_path, "--", input_path, output_path, *[f"{k}={v}" for k, v in extra_args.items()]
       - Uses asyncio.create_subprocess_exec with stdout/stderr PIPE
       - Applies asyncio.wait_for with timeout_seconds (default 120)
       - Returns {"output_path": output_path, "stdout": stdout_str, "exit_code": str(returncode)}
       - Raises RuntimeError on non-zero exit code with stderr included
    3. `get_tool_spec(self) -> ToolSpec` -- name="blender", requires_approval=False (Blender is free/local)
    4. `validate(self, **kwargs) -> bool` -- checks script name is in known scripts set

    Blender bpy scripts (these run INSIDE Blender's Python, not Jarvis Python):

    **optimize_mesh.py**: Reads sys.argv after "--". Imports FBX/GLB, applies decimate modifier (ratio from args, default 0.5), recalculates normals, smart UV project, exports FBX. Uses bpy.ops and bpy.context.

    **generate_lod.py**: Imports model, creates 3 LOD copies at 100%/50%/25% poly count via decimate modifier, exports each as separate FBX named {base}_LOD0.fbx, {base}_LOD1.fbx, {base}_LOD2.fbx.

    **generate_geometry.py**: Creates procedural geometry based on geometry_type arg:
      - "cube" / "box": bpy.ops.mesh.primitive_cube_add with dimensions
      - "plane" / "terrain": subdivided plane with noise displacement
      - "cylinder" / "pillar": cylinder with configurable segments
      Exports as FBX.

    All subprocess calls mocked in tests. Scripts tested structurally (arg parsing, import paths) not via real Blender.
