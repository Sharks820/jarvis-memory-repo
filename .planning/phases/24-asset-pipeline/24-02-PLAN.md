  <action>
    Create AssetTool class:

    1. `__init__(self, unity_tool: Any, tripo_tool: Any = None, blender_tool: Any = None)` -- takes references to other tools for delegation. tripo_tool and blender_tool are optional (graceful degradation if not available).

    2. `async execute(self, *, action: str, **kwargs: Any) -> dict[str, Any]` -- dispatches based on action:

       **"import_model"** (path: str):
       - Validates path starts with "Assets/JarvisGenerated/"
       - Calls unity_tool to set ModelImporter settings via bridge:
         - Scale factor: 1.0
         - Import materials: true
         - Generate lightmap UVs: true
         - Mesh compression: medium
       - Calls AssetDatabase.ImportAsset(path)
       - Returns {"imported": path, "type": "model"}

       **"import_texture"** (path: str):
       - Validates path inside jail
       - Sets TextureImporter: sRGB=true, maxTextureSize=2048, textureCompression=Normal, mipmapEnabled=true
       - Calls AssetDatabase.ImportAsset(path)
       - Returns {"imported": path, "type": "texture"}

       **"import_audio"** (path: str):
       - Validates path inside jail
       - Sets AudioImporter: loadInBackground=true, compressionFormat=Vorbis, quality=0.7
       - Calls AssetDatabase.ImportAsset(path)
       - Returns {"imported": path, "type": "audio"}

       **"route"** (description: str):
       - Classifies description into "tripo" or "blender" using keyword matching:
         - TRIPO keywords: character, creature, animal, organic, prop, weapon, furniture, plant, tree, food, clothing, accessory, vehicle, monster, NPC, human, body
         - BLENDER keywords: wall, floor, ceiling, terrain, landscape, building, architecture, road, bridge, stairs, column, pillar, fence, platform, tile, block, primitive, box, cylinder, sphere, geometric
       - Default: "tripo" (organic is more common for game assets)
       - Returns {"tool": "tripo"|"blender", "reason": str}

       **"generate"** (description: str, output_dir: str, format: str = "fbx"):
       - Calls route logic to pick tool
       - If tripo: calls tripo_tool.execute(prompt=description, format=format)
       - If blender: infers geometry_type from description, calls blender_tool.execute(script="generate_geometry", ...)
       - Then calls self.execute(action="import_model", path=result_path) to import into Unity
       - Returns {"model_path": str, "tool_used": str, "imported": bool}

       **"batch_import"** (paths: list[str]):
       - Calls StartAssetEditing via unity_tool
       - Imports each path with appropriate import settings (detect type by extension: .fbx/.glb=model, .png/.jpg/.tga=texture, .wav/.mp3/.ogg=audio)
       - Calls StopAssetEditing
       - Returns {"imported_count": int, "paths": list[str]}

    3. `get_tool_spec(self) -> ToolSpec` -- name="asset", requires_approval=False (import itself is safe; tripo credit spend is gated by TripoTool's own approval flag)

    4. `validate(self, **kwargs) -> bool` -- checks action is one of known actions

    All UnityTool calls mocked in tests. Tripo/Blender tool calls mocked. No real Unity bridge or API calls.

    IMPORTANT: The _send_rpc or call pattern from UnityTool is used for bridge communication. Since AssetTool does not directly use WebSocket, it delegates to UnityTool which handles the bridge protocol. For import settings, AssetTool constructs the appropriate JSON-RPC params dict and passes to unity_tool.
