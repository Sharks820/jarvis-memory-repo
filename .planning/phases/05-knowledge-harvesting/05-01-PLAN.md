  <action>
Create the `engine/src/jarvis_engine/harvesting/` package:

**providers.py** -- Provider abstraction and concrete implementations:
- `HarvesterProvider` base class with:
  - Constructor: `name: str`, `api_key: str`, `base_url: str`, `model: str`, `input_cost_per_mtok: float`, `output_cost_per_mtok: float`
  - Lazy-create `OpenAI` client on first `query()` call (not in constructor) to avoid import-time SDK dependency. Use `from openai import OpenAI` inside method body.
  - `query(topic: str, system_prompt: str, max_tokens: int = 2048) -> HarvestResult` method that calls `client.chat.completions.create()`, extracts text and usage, calculates cost, returns `HarvestResult`
  - `is_available() -> bool` class method that checks if the API key env var is set
- `MiniMaxProvider(HarvesterProvider)`:
  - Reads `MINIMAX_API_KEY` env var
  - `base_url="https://api.minimax.io/v1"`, `model="MiniMax-M2.5"`, pricing `$0.30/$1.20` per Mtok
- `KimiProvider(HarvesterProvider)`:
  - Reads `KIMI_API_KEY` env var
  - `base_url="https://api.moonshot.cn/v1"`, `model="kimi-k2.5"`, pricing `$0.60/$2.50` per Mtok
- `KimiNvidiaProvider(HarvesterProvider)`:
  - Reads `NVIDIA_API_KEY` env var
  - `base_url="https://integrate.api.nvidia.com/v1"`, `model="moonshotai/kimi-k2-5"`, pricing `$0.00/$0.00` (free)
  - Override `query()` to pass `extra_body={"thinking": {"type": "disabled"}}` for instant mode
- `GeminiProvider` (NOT inheriting from HarvesterProvider -- uses different SDK):
  - `name="gemini"`, `model="gemini-2.5-flash"`, pricing `$0.00/$0.00`
  - Reads `GEMINI_API_KEY` env var
  - Lazy-create `genai.Client(api_key=key)` using `from google import genai` inside method body
  - `query()` calls `client.models.generate_content(model=self.model, contents=...)`, extracts `response.text`
  - Tries to read `response.usage_metadata` for token counts; falls back to `len(text) // 4` character estimation if unavailable

All providers must handle missing API keys gracefully: `__init__` sets `self._available = False` if env var is empty/missing. `query()` raises `RuntimeError("Provider {name} not configured")` if called when unavailable.

**harvester.py** -- Orchestrator and dataclasses:
- `HarvestResult` dataclass (non-frozen): `provider: str`, `text: str`, `model: str`, `input_tokens: int = 0`, `output_tokens: int = 0`, `cost_usd: float = 0.0`
- `HarvestCommand` frozen dataclass: `topic: str`, `providers: list[str] | None = None` (None = all available), `max_tokens: int = 2048`
- `KnowledgeHarvester` class:
  - Constructor: `providers: list`, `pipeline: EnrichedIngestPipeline | None`, `cost_tracker: CostTracker | None`
  - `SYSTEM_PROMPT` class constant: instructs models to output factual, structured knowledge statements. Avoids opinions, speculation, conversational filler.
  - `harvest(cmd: HarvestCommand) -> dict`: iterates over requested providers (or all available), calls `provider.query()`, logs cost via `cost_tracker.log()` with `route_reason=f"harvest:{cmd.topic}"`, ingests text via `pipeline.ingest(source=f"harvest:{provider.name}", kind="semantic", task_id=f"harvest:{cmd.topic}", content=result.text, tags=["harvested", provider.name, topic_tag])`, returns `{"topic": ..., "results": [{"provider": ..., "status": ..., "records_created": ..., "cost_usd": ...}]}`
  - Wraps each provider call in try/except -- individual provider failures do not halt the harvest
  - Harvested content enters pipeline at default confidence (0.72) since the pipeline handles that. The fact that it comes from an external LLM source is tracked via the `source=f"harvest:{name}"` field and `["harvested"]` tag.
  - `available_providers() -> list[str]`: returns names of providers with valid API keys

**__init__.py** -- Package exports:
- Export: `KnowledgeHarvester`, `HarvestCommand`, `HarvestResult`, `MiniMaxProvider`, `KimiProvider`, `KimiNvidiaProvider`, `GeminiProvider`, `HarvesterProvider`

**pyproject.toml** -- Add dependencies:
- Add `openai>=1.0.0` and `google-genai>=1.0.0` to the `dependencies` list in pyproject.toml. Keep all existing dependencies unchanged.
