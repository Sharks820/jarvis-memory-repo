**session_ingestors.py** -- Parse Claude Code and Codex JSONL session files:

`ClaudeCodeIngestor`:
- `SESSION_BASE = Path.home() / ".claude" / "projects"` (resolves correctly on Windows 11 via `Path.home()`)
- Also check `CLAUDE_CONFIG_DIR` env var as override for session base path
- `find_sessions(project_path: str | None = None) -> list[Path]`: glob for `*.jsonl` files under SESSION_BASE (optionally scoped to project). Return sorted by modification time (newest first).
- `ingest_session(session_path: Path) -> list[str]`: Read JSONL line-by-line. For each line, parse JSON. Filter for `type == "assistant"` entries. Extract text content from `message.content` (handles both string and list-of-blocks format). Skip short text blocks (<100 chars -- these are tool outputs, not knowledge). Return list of extracted text strings.

`CodexIngestor`:
- `SESSION_BASE = Path(os.environ.get("CODEX_HOME", str(Path.home() / ".codex"))) / "sessions"`
- `find_sessions(days_back: int = 7) -> list[Path]`: glob for `rollout-*.jsonl` files. Return sorted by modification time.
- `ingest_session(session_path: Path) -> list[str]`: Same JSONL parsing pattern. Extract text content from entries with substantial text (>100 chars). Handle both string and list-of-blocks content format.

Both ingestors must handle:
- `FileNotFoundError` for missing session dirs (return empty list)
- `json.JSONDecodeError` for malformed lines (skip the line, continue)
- `PermissionError` (log warning, return empty list)

**budget.py** -- Per-provider budget enforcement:

`BudgetManager`:
- Constructor: `db_path: Path` -- Uses the SAME SQLite database as CostTracker (the jarvis_memory.db or cost tracker DB). Creates `harvest_budgets` and `harvest_spend` tables if they don't exist.
- Schema:
  ```sql
  CREATE TABLE IF NOT EXISTS harvest_budgets (
      provider TEXT NOT NULL,
      period TEXT NOT NULL,  -- 'daily' or 'monthly'
      limit_usd REAL NOT NULL,
      limit_requests INTEGER NOT NULL DEFAULT 0,
      PRIMARY KEY (provider, period)
  );
  CREATE TABLE IF NOT EXISTS harvest_spend (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      provider TEXT NOT NULL,
      ts TEXT NOT NULL DEFAULT (datetime('now')),
      cost_usd REAL NOT NULL DEFAULT 0.0,
      request_count INTEGER NOT NULL DEFAULT 1,
      topic TEXT NOT NULL DEFAULT ''
  );
  CREATE INDEX IF NOT EXISTS idx_harvest_spend_provider_ts ON harvest_spend(provider, ts);
  ```
- Uses WAL mode + threading.Lock (same pattern as CostTracker and MemoryEngine)
- `set_budget(provider: str, period: str, limit_usd: float, limit_requests: int = 0)`: INSERT OR REPLACE into harvest_budgets
- `can_spend(provider: str) -> bool`: Check both daily and monthly budgets. Query harvest_spend for today's total (WHERE ts >= date('now')) and this month's total (WHERE ts >= date('now', 'start of month')). Compare against limits. Return True if both under limit (or no limit configured for that provider). For request count limits (Gemini), also check request_count sum.
- `record_spend(provider: str, cost_usd: float, topic: str = "")`: INSERT into harvest_spend
- `get_spend_summary(provider: str | None = None, days: int = 30) -> dict`: Return per-provider spend breakdown for the last N days
- Default budgets (set during `_init_schema` only if table is empty):
  - `minimax`: daily $1.00, monthly $10.00
  - `kimi`: daily $1.00, monthly $10.00
  - `gemini`: daily $0.00 (free) with `limit_requests=50` per day (reserve capacity)
  - `kimi_nvidia`: daily $0.00 (free) with `limit_requests=100` per day

**harvest_commands.py** -- Typed command dataclasses:

```python
@dataclass(frozen=True)
class HarvestTopicCommand:
    """Harvest knowledge about a topic from external AI sources."""
    topic: str
    providers: list[str] | None = None  # None = all available
    max_tokens: int = 2048

@dataclass
class HarvestTopicResult:
    topic: str
    results: list[dict]
    return_code: int = 0

@dataclass(frozen=True)
class IngestSessionCommand:
    """Ingest knowledge from a Claude Code or Codex session file."""
    source: str  # "claude" or "codex"
    session_path: str | None = None  # None = discover recent sessions
    project_path: str | None = None  # For Claude Code: scope to project

@dataclass
class IngestSessionResult:
    source: str
    sessions_processed: int
    records_created: int
    return_code: int = 0

@dataclass(frozen=True)
class HarvestBudgetCommand:
    """View or set harvest budget limits."""
    action: str = "status"  # "status", "set"
    provider: str | None = None
    period: str | None = None  # "daily" or "monthly"
    limit_usd: float | None = None
    limit_requests: int | None = None

@dataclass
class HarvestBudgetResult:
    summary: dict
    return_code: int = 0
```

**harvest_handlers.py** -- Command handlers:

`HarvestHandler`:
- Constructor: `harvester: KnowledgeHarvester | None = None`
- `handle(cmd: HarvestTopicCommand) -> HarvestTopicResult`: If harvester is None, return error result. Otherwise call `harvester.harvest(HarvestCommand(topic=cmd.topic, providers=cmd.providers, max_tokens=cmd.max_tokens))` and wrap result.

`IngestSessionHandler`:
- Constructor: `pipeline: EnrichedIngestPipeline | None = None`
- `handle(cmd: IngestSessionCommand) -> IngestSessionResult`: Create appropriate ingestor (ClaudeCodeIngestor or CodexIngestor based on `cmd.source`). Find sessions via `find_sessions()`. For each session, call `ingest_session()` to get text chunks, then call `pipeline.ingest(source=f"session:{cmd.source}", kind="semantic", task_id=f"session:{session_path.name}", content=text, tags=["session", cmd.source])` for each chunk. Count total records created. If pipeline is None, return error result.

`HarvestBudgetHandler`:
- Constructor: `budget_manager: BudgetManager | None = None`
- `handle(cmd: HarvestBudgetCommand) -> HarvestBudgetResult`: For "status" action, call `budget_manager.get_spend_summary()`. For "set" action, call `budget_manager.set_budget()`. If budget_manager is None, return error result.

All handlers follow the existing pattern: accept optional dependencies in constructor, return typed result dataclass, graceful degradation when dependencies are None.
