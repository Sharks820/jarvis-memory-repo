## User-Critical Outcomes

1. Preserve conversation context when swapping LLM providers (no reset/new-conversation behavior).
2. Improve date/time grounding so reasoning responses reflect current day and timezone.
3. Improve voice UX and realtime listening trust (clear "listening now" UI signal + lower latency).
4. Upgrade activity telemetry from "accepted/completed" to live, inspectable execution traces.
5. Upgrade learning mission UX with true status/progress, live activity, cancellation, restart, and scheduling controls.
6. Add autonomous self-diagnosis + approval-gated auto-fix flow.
7. Improve spoken/web citation rendering so voice reads concise page titles instead of full raw URLs.
8. Begin memory hygiene loop: identify junk context, summarize/archive safely, keep high-value memory salient.
9. Use desloppify as quality gate and work queue until strict score trends to 100.

## Strategy

1. **Continuity before expansion:** eliminate provider-swap context loss and time-grounding drift first.
2. **Trustable UX:** listening indicator + mission progress bars must reflect backend truth in near-real time.
3. **Actionable observability:** users must see what Jarvis is doing now, not only lifecycle edges.
4. **Safe autonomy:** Jarvis can propose/prepare code fixes but only apply after explicit owner approval.
5. **Anti-regression gates:** every tranche ships with contract tests and telemetry assertions.
6. **Debt-to-100 loop:** run desloppify in repeated batches (objective + subjective) with attestable fixes.

## Delivery Tasks

### Task A: Cross-LLM Context Continuity Contracts

- Enforce a provider-agnostic conversation state object (`session_id`, `checkpoint_id`, `rolling_summary`, `anchor_entities`).
- Add hard contract tests for model swap continuity:
  - `cloud -> local`, `local -> cloud`, `fallback after provider failure`.
- Add explicit telemetry events for continuity reconstruction success/failure.
- Add regression gate: no swap may drop prior user intent, entities, or unresolved mission steps.

### Task B: Time/Date Grounding Reliability

- Inject canonical runtime clock (`current_date`, `timezone`, `epoch`) into reasoning context.
- Add stale-time detector: flag responses citing old date context when session clock advanced.
- Add tests for date-sensitive prompts and reasoning chains.

### Task C: Voice Realtime + Listening Indicator UX

- Surface explicit listening states in UI:
  - `idle`, `arming`, `listening`, `processing`, `speaking`, `error`.
- Add animated listening indicator (spinner/ring/wave) with accessibility labels.
- Emit low-latency voice pipeline telemetry (`vad_start`, `vad_end`, partial transcript lag, final transcript lag).
- Add acceptance SLOs for median and p95 capture-to-transcript times.

### Task D: Mission and Activity Transparency

- Expand activity stream payload:
  - `stage`, `substep`, `elapsed_ms`, `progress_pct`, `artifact_count`, `current_action`.
- Add "Now working on" live section in dashboard.
- Add learning mission status panel with:
  - active/inactive indicator
  - per-mission progress bar
  - latest learned items snapshot
  - stop/restart/timer-schedule controls
- Replace generic success text with structured mission status output.

### Task E: Autonomous Self-Diagnosis + Approval-Gated Auto-Fix

- Add diagnostics command path that runs targeted health checks and returns ranked issues.
- Add patch proposal pipeline:
  - generate diff
  - explain risk/scope/tests
  - request explicit owner approval
  - apply + verify + log provenance on approval.
- Persist approval decisions and patch audit trail for rollback/forensics.

### Task F: Memory Hygiene and Signal Extraction

- Introduce memory quality classifier tiers: `high-signal`, `contextual`, `ephemeral`, `junk`.
- Add automated cleanup candidates list with safe preview before deletion/archive.
- Add anti-loss guardrails for personal facts, commitments, and active mission context.

### Task G: Citation/Voice Link Summarization

- Convert raw URLs to concise title-first references in speech/UI.
- Preserve full URL in expandable details for auditability.
- Add tests to ensure TTS output does not read full raw links by default.

### Task H: Desloppify 100 Program

- Install/verify full scan dependencies (`bandit`, optional language scanners).
- Run sequence each cycle:
  1. `scan`
  2. `next`/`show` triage
  3. fix + tests
  4. resolve with attestation
  5. rescan
- Treat subjective review batches as first-class work, not optional.
- Track strict score trendline in `.planning/STATE.md` until 100 target.

- Provider swaps preserve context and unresolved goals in all tested routes.
- Date-aware responses stay synchronized with runtime clock.
- UI visibly indicates listening state and transitions correctly.
- Learning missions expose live status/progress and controllable lifecycle.
- Activity feed shows actionable live execution details.
- Auto-fix can diagnose and propose patches, but only applies after explicit approval.
- Desloppify strict score trend is continuously improving with no score-gaming behavior.
