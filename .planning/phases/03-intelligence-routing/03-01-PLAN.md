<objective>
Build the ModelGateway foundation: a unified LLM completion interface wrapping the Anthropic SDK and Ollama Python client, with automatic fallback chains and per-query cost tracking in SQLite.

Purpose: This is the core infrastructure that all intelligence routing builds on. The gateway provides a single `complete()` method that dispatches to the correct provider, handles failures gracefully, and logs costs -- so the rest of the system never needs to know which model answered.

Output: `gateway/` package with ModelGateway, CostTracker, pricing table, GatewayResponse dataclass, and comprehensive tests.
</objective>

    1. Add `"anthropic>=0.81.0"` and `"ollama>=0.4.0"` to `dependencies` in `engine/pyproject.toml`. Keep existing deps. Run `pip install -e engine/` to install.

    2. Create `engine/src/jarvis_engine/gateway/pricing.py`:
       - Define `PRICING: dict[str, tuple[float, float]]` mapping model-name prefixes to (input_cost_per_mtok, output_cost_per_mtok).
       - Include entries: `"claude-opus"` -> (15.0, 75.0), `"claude-sonnet"` -> (3.0, 15.0), `"claude-haiku"` -> (0.80, 4.0). These are current Feb 2026 Anthropic pricing.
       - Ollama models cost 0.0 (local). Default fallback cost is 0.0 for unknown models.
       - Add a `calculate_cost(model: str, input_tokens: int, output_tokens: int) -> float` function that matches the model string against PRICING keys using `startswith` and returns the USD cost. Return 0.0 for unrecognized models (local models).

    3. Create `engine/src/jarvis_engine/gateway/costs.py`:
       - `CostTracker` class with `__init__(self, db_path: Path)`.
       - On init: open sqlite3 connection with `check_same_thread=False`, set `PRAGMA journal_mode=WAL` and `PRAGMA busy_timeout=5000`, call `_init_schema()`.
       - `_init_schema()`: CREATE TABLE IF NOT EXISTS `query_costs` with columns: `id INTEGER PRIMARY KEY AUTOINCREMENT`, `ts TEXT NOT NULL DEFAULT (datetime('now'))`, `model TEXT NOT NULL`, `provider TEXT NOT NULL`, `input_tokens INTEGER NOT NULL DEFAULT 0`, `output_tokens INTEGER NOT NULL DEFAULT 0`, `cost_usd REAL NOT NULL DEFAULT 0.0`, `route_reason TEXT NOT NULL DEFAULT ''`, `fallback_used INTEGER NOT NULL DEFAULT 0`, `query_hash TEXT NOT NULL DEFAULT ''`. Create indexes on `ts` and `model`.
       - Use a `threading.Lock()` for write serialization (same pattern as MemoryEngine._write_lock).
       - `log(model, input_tokens, output_tokens, cost_usd=None, route_reason="", fallback_used=False, query_hash="")`: If cost_usd is None, calculate via `pricing.calculate_cost()`. INSERT row under the write lock. Commit.
       - `summary(days=30) -> dict`: SELECT model, COUNT, SUM(input_tokens), SUM(output_tokens), SUM(cost_usd) grouped by model for the last N days. Return dict with `period_days`, `models` list, and `total_cost_usd`.
       - `close()`: Close the DB connection.

    4. Create `engine/src/jarvis_engine/gateway/models.py`:
       - Import `Anthropic, APIConnectionError, APIStatusError, RateLimitError` from `anthropic`.
       - Import `Client as OllamaClient, ResponseError` from `ollama`.
       - Define `GatewayResponse` dataclass (NOT frozen): `text: str`, `model: str`, `provider: str`, `input_tokens: int = 0`, `output_tokens: int = 0`, `cost_usd: float = 0.0`, `fallback_used: bool = False`, `fallback_reason: str = ""`.
       - Define `ModelGateway` class:
         - `__init__(self, anthropic_api_key: str | None = None, ollama_host: str = "http://127.0.0.1:11434", cost_tracker: CostTracker | None = None)`.
         - If `anthropic_api_key` is provided, create `self._anthropic = Anthropic(api_key=anthropic_api_key)`. Otherwise `self._anthropic = None`. If key is None, log a warning: "No Anthropic API key configured -- operating in local-only mode".
         - Create `self._ollama = OllamaClient(host=ollama_host)`.
         - Store `self._cost_tracker = cost_tracker`.
         - `_resolve_provider(model: str) -> str`: Return `"anthropic"` if model starts with `"claude-"` and `self._anthropic is not None`. Return `"ollama"` otherwise.
         - `complete(messages: list[dict[str, str]], model: str = "claude-sonnet-4-5-20250929", max_tokens: int = 1024, route_reason: str = "") -> GatewayResponse`:
           - Call `_resolve_provider(model)` to determine provider.
           - If provider is `"anthropic"`, call `_call_anthropic()`. Wrap in try/except for `APIConnectionError`, `APIStatusError`, `RateLimitError` -- on any of these, call `_fallback_to_ollama()`.
           - If provider is `"ollama"`, call `_call_ollama()`.
           - After getting response, if `self._cost_tracker`, call `self._cost_tracker.log(...)` with the response metadata.
           - Return the GatewayResponse.
         - `_call_anthropic(messages, model, max_tokens) -> GatewayResponse`: Use `self._anthropic.messages.create(model=model, max_tokens=max_tokens, messages=messages)`. Extract `response.content[0].text`, `response.usage.input_tokens`, `response.usage.output_tokens`. Calculate cost via `pricing.calculate_cost()`. Return GatewayResponse.
         - `_call_ollama(messages, model, max_tokens) -> GatewayResponse`: Use `self._ollama.chat(model=model, messages=messages)`. Extract `response.message.content`. Return GatewayResponse with input/output_tokens=0 and cost_usd=0.0.
         - `_fallback_to_ollama(messages, max_tokens, reason: str) -> GatewayResponse`: Use configurable fallback model (default `"qwen3:14b"`, overridable via `JARVIS_LOCAL_MODEL` env var). Try `self._ollama.chat()`. On success return GatewayResponse with `fallback_used=True` and `fallback_reason` set. On `ConnectionError` or `ResponseError`, return GatewayResponse with `text=""`, `provider="none"`, `fallback_used=True`, and full error reason.
         - `check_ollama() -> bool`: Try `self._ollama.list()` in a try/except. Return True if reachable, False otherwise.
         - `check_anthropic() -> bool`: Return `self._anthropic is not None`.

    5. Create `engine/src/jarvis_engine/gateway/__init__.py`:
       - Export: `ModelGateway`, `GatewayResponse`, `CostTracker`.
       - Use lazy imports (import inside the __init__.py at module level is fine here since these are all local modules).

    IMPORTANT: Do NOT import from `anthropic` or `ollama` at package __init__ level -- keep imports inside the classes so the package can be imported even if the SDKs are not yet installed. The gateway/models.py should do the imports at the top of the file (that file will only be loaded when ModelGateway is actually used).
