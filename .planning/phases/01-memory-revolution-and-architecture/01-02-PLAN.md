<objective>
Build the SQLite + FTS5 + sqlite-vec memory engine with hybrid search and tiered storage. This creates the real database backend that replaces JSONL flat files for all memory operations. The engine supports concurrent access via WAL mode, keyword search via FTS5, semantic search via sqlite-vec, and combined retrieval via Reciprocal Rank Fusion.

Purpose: This is Jarvis's new brain. Without a queryable database with semantic search, the assistant can only find things by exact keyword match. The hybrid search engine gives Jarvis the ability to find relevant memories by meaning -- asking "what medications do I take" will find records about prescriptions even if they never use the word "medications."

Output: MemoryEngine class, hybrid_search module, TierManager, updated pyproject.toml with new dependencies, and comprehensive tests.
</objective>

    ```sql
    CREATE TABLE IF NOT EXISTS records (
        record_id TEXT PRIMARY KEY,
        ts TEXT NOT NULL,
        source TEXT NOT NULL,
        kind TEXT NOT NULL,
        task_id TEXT NOT NULL DEFAULT '',
        branch TEXT NOT NULL DEFAULT 'general',
        tags TEXT NOT NULL DEFAULT '[]',
        summary TEXT NOT NULL,
        content_hash TEXT NOT NULL,
        confidence REAL NOT NULL DEFAULT 0.72,
        tier TEXT NOT NULL DEFAULT 'warm',
        access_count INTEGER NOT NULL DEFAULT 0,
        last_accessed TEXT NOT NULL DEFAULT '',
        created_at TEXT NOT NULL DEFAULT (datetime('now'))
    );

    CREATE UNIQUE INDEX IF NOT EXISTS idx_content_hash ON records(content_hash);

    -- Standalone FTS5 (content='', contentless_delete=1) -- simpler than external content
    CREATE VIRTUAL TABLE IF NOT EXISTS fts_records
        USING fts5(record_id, summary, content='', contentless_delete=1);

    -- sqlite-vec for embedding vectors
    CREATE VIRTUAL TABLE IF NOT EXISTS vec_records
        USING vec0(record_id TEXT PRIMARY KEY, embedding float[768]);

    -- Facts table (preserved from brain_memory.py structure)
    CREATE TABLE IF NOT EXISTS facts (
        key TEXT PRIMARY KEY,
        value TEXT NOT NULL,
        confidence REAL NOT NULL DEFAULT 0.0,
        locked INTEGER NOT NULL DEFAULT 0,
        updated_utc TEXT NOT NULL,
        sources TEXT NOT NULL DEFAULT '[]',
        history TEXT NOT NULL DEFAULT '[]'
    );

    -- Schema version tracking
    CREATE TABLE IF NOT EXISTS schema_version (
        version INTEGER PRIMARY KEY,
        applied_at TEXT NOT NULL DEFAULT (datetime('now'))
    );
    INSERT OR IGNORE INTO schema_version(version) VALUES (1);
    ```

    Methods:
    - `insert_record(self, record: dict, embedding: list[float] | None = None) -> bool`: Insert into records table, fts_records, and vec_records (if embedding provided and `self._vec_available`) in a single transaction under write lock. Use `INSERT OR IGNORE` to handle dedup via content_hash UNIQUE constraint. Serialize embedding via `struct.pack(f"{len(embedding)}f", *embedding)`. Return True if inserted, False if duplicate. NOTE (Codex finding): The content_hash UNIQUE constraint is per-record, not per-document. When the ingestion pipeline (Plan 03) chunks a document into N pieces, each chunk gets its OWN content_hash (SHA-256 of the chunk text), so all N chunks can be inserted. The dedup catches re-ingestion of the same chunk, not chunks from the same document.
    - `get_record(self, record_id: str) -> dict | None`: Fetch single record by ID.
    - `search_fts(self, query: str, limit: int = 30) -> list[tuple[str, float]]`: FTS5 search returning (record_id, rank) pairs.
    - `search_vec(self, query_embedding: list[float], limit: int = 30) -> list[tuple[str, float]]`: sqlite-vec KNN search returning (record_id, distance) pairs. Serialize query_embedding with struct.pack. If `self._vec_available` is False, return empty list (graceful degradation).
    - `update_access(self, record_id: str)`: Increment access_count, set last_accessed to now(). Under write lock.
    - `update_tier(self, record_id: str, tier: str)`: Update tier column. Under write lock.
    - `get_all_record_ids(self) -> list[str]`: List all record IDs (for tier management).
    - `count_records(self) -> int`: Return total record count.
    - `close(self)`: Close db connection.

    CRITICAL: For FTS5 standalone mode, when inserting use `INSERT INTO fts_records(record_id, summary) VALUES (?, ?)` -- NOT using rowid from records table since content='' means FTS5 manages its own storage.

    CRITICAL: For sqlite-vec, vectors MUST be serialized as bytes via `struct.pack(f"{len(vec)}f", *vec)` for both INSERT and MATCH operations.

    **Step 1: Hybrid search module** (engine/src/jarvis_engine/memory/search.py)

    Create `hybrid_search()` function that combines FTS5 keyword + sqlite-vec semantic + recency decay:

    ```python
    def hybrid_search(
        engine: MemoryEngine,
        query: str,
        query_embedding: list[float],
        k: int = 10,
        rrf_k: int = 60,
        recency_weight: float = 0.3,
    ) -> list[dict]:
    ```

    Algorithm:
    1. **FTS5 keyword search**: Call `engine.search_fts(query, limit=k*3)` to get (record_id, rank) pairs
    2. **sqlite-vec KNN search**: Call `engine.search_vec(query_embedding, limit=k*3)` to get (record_id, distance) pairs
    3. **Reciprocal Rank Fusion**: Combine scores:
       - For each FTS5 result at position i: `score[rid] += 1.0 / (rrf_k + i + 1)`
       - For each vec result at position i: `score[rid] += 1.0 / (rrf_k + i + 1)`
    4. **Recency boost**: For each scored record:
       - Compute recency as exponential decay: `recency = exp(-age_hours / 168)` (168 = 7 days half-life)
       - Apply: `score[rid] *= (1.0 + recency_weight * recency)`
    5. **Sort by combined score** descending, return top-k records
    6. **Update access counts** for returned records via `engine.update_access(rid)`

    Also create helper function:
    ```python
    def _recency_weight(ts_str: str) -> float:
        """Compute exponential recency decay. Returns 0.0-1.0."""
    ```
