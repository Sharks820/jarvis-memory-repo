<objective>
Decompose the monolithic main.py (45 cmd_* functions, ~1700 lines) into a Command Bus architecture with typed commands, separate handlers, and a service layer. All interfaces (CLI, mobile API, daemon) will produce Command objects dispatched through the same bus. Handlers initially delegate to existing functions via adapter shims to guarantee zero test regression.

Purpose: This is the foundational architecture change that enables all subsequent work. Without decoupled handlers, the memory engine (Plan 02) and ingestion pipeline (Plan 03) have no clean integration points. The Command Bus also enables testing handlers independently of CLI parsing.

Output: command_bus.py, commands/ directory with typed dataclasses, handlers/ directory with adapter-shim handlers, app.py bootstrap, lazy-loaded EmbeddingService, and updated main.py that dispatches through the bus. All 125+ existing tests pass.
</objective>

    Modify main.py to dispatch through the Command Bus instead of inlining logic. This is the structural pivot point.

    **Strategy:** Keep ALL `cmd_*` function signatures and names unchanged. They still exist in main.py. Tests that call `main_mod.cmd_voice_run(...)` will still work. The ONLY change is that inside each `cmd_*` function, instead of inlining logic, it:
    1. Creates the appropriate Command dataclass
    2. Calls `_get_bus().dispatch(command)`
    3. Formats the result for output (print, return code) exactly as before

    **Implementation:**

    1. Add a module-level `_bus: CommandBus | None = None` variable
    2. Add `_get_bus() -> CommandBus` function that lazily creates the bus via `create_app(repo_root())`
    3. For each of the 45 `cmd_*` functions:
       - Keep the function signature identical
       - Create the corresponding Command dataclass from the function parameters
       - Call `result = _get_bus().dispatch(command)`
       - Handle the result to produce identical output (prints, return codes)

    **Example transformation:**
    ```python
    # BEFORE
    def cmd_brain_status(as_json: bool) -> int:
        data = brain_status(repo_root())
        if as_json:
            print(json.dumps(data, indent=2))
        else:
            for b in data["branches"]:
                print(f"  {b['branch']}: {b['count']} records")
        return 0

    # AFTER
    def cmd_brain_status(as_json: bool) -> int:
        cmd = BrainStatusCommand(as_json=as_json)
        result = _get_bus().dispatch(cmd)
        if as_json:
            print(json.dumps(result.data, indent=2))
        else:
            for b in result.branches:
                print(f"  {b['branch']}: {b['count']} records")
        return 0
    ```

    **Key constraints:**
    - Every `cmd_*` function must remain importable at `jarvis_engine.main`
    - All monkeypatch targets in test_main.py must still resolve (check what tests monkeypatch and ensure those attributes still exist on the main module)
    - The `main()` function (argparse entry point) does NOT change -- it still calls cmd_* functions
    - Output format (print statements, JSON output, return codes) must be IDENTICAL to current behavior
    - Keep all existing imports in main.py -- they are still needed for output formatting

    **For complex cmd_* functions** (like cmd_daemon_run, cmd_ops_autopilot, cmd_voice_run) that contain deeply nested logic with callbacks and subprocess management:
    - Keep the complex logic in the handler, not in main.py
    - The cmd_* wrapper should create the command with all needed parameters and dispatch

    - If any test fails, the issue is in the rewiring -- fix the cmd_* wrapper to produce identical output
    - Do NOT modify existing test files -- the point is zero-regression
