    Create `BranchClassifier` class that replaces the keyword-based `_pick_branch()` from brain_memory.py:

    ```python
    BRANCH_DESCRIPTIONS = {
        "ops": "calendar scheduling meetings daily operations email organization tasks planning",
        "coding": "programming software development debugging testing code deployment build compile",
        "health": "medications prescriptions doctor appointments health pharmacy wellness exercise",
        "finance": "budget banking payments invoices expenses financial planning investment",

        "family": "children family school spouse home activities parenting",
        "communications": "phone calls text messages SMS contacts communication voicemail",
        "gaming": "video games gaming sessions steam fortnite competitive play esports",
    }
    ```

    - Constructor `__init__(self, embed_service: EmbeddingService)`:
      - Store embed_service reference
      - `self._centroids: dict[str, list[float]] | None = None` (lazy computed)
    - `_ensure_centroids(self)`:
      - If centroids is None, compute: `embed_service.embed(desc, prefix="classification")` for each branch
      - Cache in self._centroids
    - `classify(self, text_embedding: list[float], threshold: float = 0.3) -> str`:
      - Ensure centroids are computed
      - Compute cosine similarity between text_embedding and each branch centroid
      - Return branch with highest similarity if above threshold, else "general"
      - Use numpy for dot product and norm calculations

    **Step 2: Enriched ingestion pipeline** (engine/src/jarvis_engine/memory/ingest.py)

    NOTE: This is a NEW file at `memory/ingest.py`, separate from the existing `engine/src/jarvis_engine/ingest.py` which is preserved.

    Create `EnrichedIngestPipeline` class:

    Constructor `__init__(self, engine: MemoryEngine, embed_service: EmbeddingService, classifier: BranchClassifier)`:
    - Store all three dependencies

    Method `ingest(self, source: str, kind: str, task_id: str, content: str, tags: list[str] | None = None) -> list[str]`:
    Pipeline steps:
    1. **Sanitize**: Strip, truncate to 10000 chars, redact password/token patterns (reuse `_sanitize_memory_content` pattern from main.py)
    2. **Pre-flight dedup**: Compute SHA-256 of FULL content as a quick check. If content is short (single chunk) and `engine.get_record_by_hash(full_content_hash)` exists, return empty list (already stored). For multi-chunk content, per-chunk dedup happens in step 4a via the UNIQUE constraint.
    3. **Chunk**: If content > 2000 chars, split into chunks of ~1500 chars at sentence boundaries (split on `. ` or `\n`). Each chunk becomes a separate record. If <= 2000 chars, single record.
    4. **For each chunk**:
       a. Generate **per-chunk** content_hash: SHA-256 of the chunk text (NOT the original document). This is CRITICAL (Codex finding): if you hash the whole document before chunking, all chunks share the same hash, and the UNIQUE constraint on content_hash means only the first chunk inserts. Each chunk must have its own hash.
       b. Generate record_id: SHA-256 of `f"{source}|{kind}|{task_id}|{chunk_content}"` truncated to 16 hex chars
       c. Generate embedding: `embed_service.embed(chunk_content, prefix="search_document")`
       d. Classify branch: `classifier.classify(embedding)`
       e. Build record dict with all fields (record_id, ts, source, kind, task_id, branch, tags, summary=chunk_content[:200], content_hash=per_chunk_hash, confidence=0.72, tier="warm")
       f. Insert via `engine.insert_record(record, embedding)`
    5. Return list of inserted record_ids

    Method `_chunk_content(self, content: str, max_chunk: int = 1500) -> list[str]`:
    - If len(content) <= max_chunk * 1.2: return [content] (don't split if barely over)
    - Split on sentence boundaries (`. `, `.\n`, `\n\n`)
    - Accumulate sentences into chunks until approaching max_chunk
    - Return list of chunks, each <= max_chunk chars (approximately)

    **Step 1: Migration script** (engine/src/jarvis_engine/memory/migration.py)

    Create migration functions that import all existing data into SQLite:

    `migrate_brain_records(jsonl_path: Path, engine: MemoryEngine, embed_service: EmbeddingService, classifier: BranchClassifier) -> dict`:
    - Read `.planning/brain/records.jsonl` line by line
    - Count total source lines (non-empty)
    - For each valid JSON line:
      - Parse into dict
      - Generate embedding via `embed_service.embed(record["summary"], prefix="search_document")`
      - Classify branch via `classifier.classify(embedding)` (override the old keyword-based branch)
      - Build record dict matching MemoryEngine schema
      - Insert via `engine.insert_record(record, embedding)`
      - Track: inserted count, skipped count (duplicates), error count
    - For malformed JSON lines: log line number + error, increment error count, continue
    - For encoding issues: use `errors='replace'`, log warning
    - ASSERT: `inserted + skipped + errors == source_count` -- fail loudly if counts do not match
    - (Codex finding — resumable migration): Track progress with a checkpoint file at `{db_path}.migration_checkpoint.json` containing `{"file": "records.jsonl", "line_offset": N, "records_hash": "sha256_of_last_processed"}`. On startup, if checkpoint exists, skip to line_offset. On each batch of 50 records, write checkpoint. On completion, delete checkpoint. This allows migration to resume after crash without re-processing.
    - Return: `{"status": "ok", "source_count": N, "inserted": N, "skipped": N, "errors": N, "error_details": [...]}`

    `migrate_facts(facts_path: Path, engine: MemoryEngine) -> dict`:

    - Generate embeddings via `embed_service.embed()` and classify via `classifier.classify()` (Codex finding: original signature was missing these required dependencies)
