    **HmacInterceptor.kt:**
    - Implements okhttp3.Interceptor
    - Constructor takes: `signingKeyProvider: () -> String?`, `deviceIdProvider: () -> String?`
    - intercept(chain: Interceptor.Chain):
      1. Get original request
      2. Generate timestamp: current epoch seconds as string (e.g., "1708700000")
      3. Generate nonce: 32 random hex chars via UUID.randomUUID().toString().replace("-", "")
      4. Read request body into bytes (buffer the body so it can be re-read)
      5. Build signing material: "$timestamp\n$nonce\n$bodyBytes" -- MUST match desktop format exactly
         - Desktop code (mobile_api.py line 603-607): `ts_raw.encode("utf-8") + b"\n" + nonce.encode("utf-8") + b"\n" + body`
         - The body is raw bytes of the JSON payload. For GET requests with no body, use empty bytes.
      6. Compute HMAC-SHA256: Mac.getInstance("HmacSHA256"), init with SecretKeySpec(signingKey.toByteArray(Charsets.UTF_8), "HmacSHA256"), update with signing material bytes, digest and hex-encode (lowercase)
      7. Add headers to request:
         - "X-Jarvis-Timestamp": timestamp
         - "X-Jarvis-Nonce": nonce
         - "X-Jarvis-Signature": hex signature (lowercase)
         - "X-Jarvis-Device-Id": deviceId (from provider)
      8. Proceed with modified request
    - CRITICAL: If signingKeyProvider returns null (not yet bootstrapped), skip signing and proceed without headers. The /bootstrap endpoint does not require HMAC.

    **SyncManager.kt:**
    - @Singleton class injected via Hilt
    - Constructor: JarvisApiClient, JarvisDatabase (for settings), CryptoHelper
    - `suspend fun syncLoop()`: infinite loop with configurable interval (default 30 seconds, read from SettingsDao key "sync_interval_seconds")
      1. Check if desktop is reachable (apiClient.isDesktopReachable())
      2. If reachable: pull sync changes, push local changes, update "last_sync" timestamp in SettingsDao
      3. If not reachable: log, continue loop (CommandQueueProcessor handles offline commands separately)
      4. delay(intervalMs)
    - `fun isConnected(): StateFlow<Boolean>` -- exposes connectivity state as StateFlow for UI
    - `fun lastSyncTime(): StateFlow<Long?>` -- exposes last sync timestamp
    - Emit state changes so JarvisService can update notification text

    **CommandQueueProcessor.kt:**
    - @Singleton class injected via Hilt
    - Constructor: JarvisApiClient, CommandQueueDao, ConversationDao
    - `suspend fun queueCommand(text: String): Long`:
      1. Insert CommandQueueEntity with status="pending", createdAt=now
      2. Insert ConversationEntity with role="user", content=text, timestamp=now
      3. Return the command ID
      4. Immediately try to send if online (don't wait for loop)
    - `suspend fun processLoop()`: infinite loop (interval 5 seconds)
      1. Get all pending/failed commands from CommandQueueDao.getPendingCommands()
      2. For each command:
         a. Update status to "sending"
         b. Call apiClient.sendCommand(command.text)
         c. On success: update status="sent", store response, insert ConversationEntity with role="assistant" and response content
         d. On failure: call markRetry(), apply exponential backoff
      3. Exponential backoff per command: delay = min(1000 * 2^retryCount, 300000) ms (1s, 2s, 4s, 8s... capped at 5 minutes)
      4. Skip commands that have been retried more than 20 times (mark as "abandoned")
    - `fun pendingCount(): StateFlow<Int>` -- for notification badge

    **OwnerGuard.kt:**
    - @Singleton class injected via Hilt
    - Constructor: JarvisApiClient, CryptoHelper (via context)
    - `suspend fun bootstrap(context: Context, desktopUrl: String, masterPassword: String): Result<BootstrapResult>`:
      1. Save desktop URL to CryptoHelper
      2. Generate a device ID: "galaxy_s25_" + first 8 chars of Android ID (Settings.Secure.ANDROID_ID)
      3. Call apiClient.bootstrap(masterPassword, deviceId)
      4. On success (status 200):
         a. Save signing key from response to CryptoHelper.saveSigningKey()
         b. Save auth token from response to CryptoHelper.saveAuthToken()
         c. Save device ID to CryptoHelper
         d. Return success with BootstrapResult(deviceId, trusted=true)
      5. On failure: return Result.failure with descriptive error
    - `fun isBootstrapped(context: Context): Boolean` -- checks if signing key exists in CryptoHelper
    - data class BootstrapResult(val deviceId: String, val trusted: Boolean)
