The voice engine completes the full round-trip: push-to-talk -> STT -> send to desktop -> receive response -> TTS playback.

  <action>
    **VoiceState.kt:**
    - Sealed class VoiceState:
      - object Idle : VoiceState()
      - object Listening : VoiceState() (STT active, mic recording)
      - data class Transcribing(val partialText: String) : VoiceState() (partial results coming in)
      - data class Processing(val transcribedText: String) : VoiceState() (sent to desktop, waiting for response)
      - data class Speaking(val responseText: String) : VoiceState() (TTS playing response)
      - data class Error(val message: String) : VoiceState()

    **VoiceEngine.kt:**
    - @Singleton class injected via Hilt
    - Constructor: CommandQueueProcessor, Application context
    - Private fields:
      - speechRecognizer: SpeechRecognizer (lazy init via SpeechRecognizer.createSpeechRecognizer(context))
      - textToSpeech: TextToSpeech (lazy init)
      - _state: MutableStateFlow<VoiceState>(VoiceState.Idle)
      - state: StateFlow<VoiceState> (public read-only)
      - ttsSpeed: Float = 1.0f (configurable)

    - `fun startListening()`:
      1. Check SpeechRecognizer.isRecognitionAvailable(context). If not: emit Error("Speech recognition not available").
      2. Create RecognizerIntent with:
         - ACTION_RECOGNIZE_SPEECH
         - EXTRA_LANGUAGE_MODEL = LANGUAGE_MODEL_FREE_FORM
         - EXTRA_LANGUAGE = "en-US"
         - EXTRA_PARTIAL_RESULTS = true
         - EXTRA_MAX_RESULTS = 1
      3. Set RecognitionListener:
         - onReadyForSpeech: emit Listening
         - onPartialResults: emit Transcribing(partialText) from Bundle RESULTS_RECOGNITION[0]
         - onResults: get final text from Bundle RESULTS_RECOGNITION[0], call processCommand(text)
         - onError: map error code to message (ERROR_NO_MATCH -> "Didn't catch that", ERROR_NETWORK -> "Network error", etc.), emit Error
      4. speechRecognizer.startListening(intent)
      5. Emit Listening state

    - `fun stopListening()`:
      - speechRecognizer.stopListening()
      - If state is Listening or Transcribing, emit Idle

    - `private suspend fun processCommand(text: String)`:
      1. Emit Processing(text)
      2. Call commandQueueProcessor.queueCommand(text) -- this saves to DB and tries to send
      3. Wait for response: poll CommandQueueDao for the command's response (check every 500ms, timeout after 30s)
         - Alternative approach: queueCommand returns a Deferred or has a callback. Since CommandQueueProcessor already inserts the response into ConversationEntity, we can observe ConversationDao for the latest assistant message after our command.
      4. When response received: emit Speaking(responseText), call speakResponse(responseText)
      5. On timeout: emit Speaking("I've queued your command, but the desktop hasn't responded yet. I'll let you know when it does.")
      6. On error: emit Error(message)

    - `private fun speakResponse(text: String)`:
      1. Initialize TextToSpeech if not already:
         - TextToSpeech(context) { status -> if (status == TextToSpeech.SUCCESS) set language to Locale.UK (British persona!) }
      2. Set speech rate: tts.setSpeechRate(ttsSpeed)
      3. Set utterance progress listener to emit Idle when done speaking
      4. tts.speak(text, TextToSpeech.QUEUE_FLUSH, null, "jarvis_response")
      5. Emit Speaking(text)

    - `fun cancelSpeaking()`:
      - tts.stop()
      - Emit Idle

    - `fun destroy()`:
      - speechRecognizer.destroy()
      - textToSpeech.shutdown()

    - `fun setTtsSpeed(speed: Float)`: update ttsSpeed, clamp to 0.5-2.0 range

    IMPORTANT NOTES:
    - SpeechRecognizer MUST be created and called from main thread. Use Dispatchers.Main for all speech operations.
    - TextToSpeech locale should be Locale.UK to match the British butler persona from the desktop engine.
    - The voice round-trip flow: User taps mic -> Listening -> user speaks -> Transcribing -> user stops -> Processing -> desktop responds -> Speaking -> response plays -> Idle
    - Push-to-talk from notification: The JarvisService notification "Talk to Jarvis" action button should send a broadcast/intent that VoiceEngine picks up. Add a BroadcastReceiver or use a PendingIntent that starts MainActivity with an extra "voice_command" flag, which then calls voiceEngine.startListening().
  </action>
