**knowledge/__init__.py:** Export KnowledgeGraph, FactExtractor, FactTriple.

**knowledge/graph.py -- KnowledgeGraph class:**
- Constructor takes a MemoryEngine instance (uses its `_db` connection and `_write_lock` for thread safety).
- `_ensure_schema()` creates three tables in the existing SQLite DB (same file as memory records):
  - `kg_nodes`: node_id TEXT PK, label TEXT NOT NULL, node_type TEXT DEFAULT 'fact', confidence REAL DEFAULT 0.5, locked INTEGER DEFAULT 0, locked_at TEXT, locked_by TEXT, sources TEXT DEFAULT '[]' (JSON array), history TEXT DEFAULT '[]' (JSON array), created_at TEXT, updated_at TEXT. Indexes on node_type and locked.
  - `kg_edges`: edge_id INTEGER PK AUTOINCREMENT, source_id TEXT NOT NULL, target_id TEXT NOT NULL, relation TEXT NOT NULL, confidence REAL DEFAULT 0.5, source_record TEXT, created_at TEXT. Foreign keys to kg_nodes. Indexes on source_id, target_id, relation. UNIQUE index on (source_id, target_id, relation).
  - `kg_contradictions`: contradiction_id INTEGER PK AUTOINCREMENT, node_id TEXT NOT NULL FK, existing_value TEXT, incoming_value TEXT, existing_confidence REAL, incoming_confidence REAL, incoming_source TEXT, record_id TEXT, status TEXT DEFAULT 'pending', resolved_at TEXT, resolution TEXT, created_at TEXT. Indexes on status and node_id.
- Bump schema_version to 2: `INSERT OR IGNORE INTO schema_version(version) VALUES (2)`.
- `to_networkx() -> nx.DiGraph`: Load all nodes and edges from SQLite, build and return a fresh NetworkX DiGraph. Do NOT cache the graph -- rebuild every call per research guidance.
- `add_fact(node_id, label, confidence, source_record, node_type='fact') -> bool`: Atomic lock-safe insert/update. If node exists AND locked AND label differs, quarantine via `_quarantine_contradiction()` and return False. If node exists AND unlocked, update with MAX(confidence) and append source to sources list (cap at 50 sources). If node does not exist, INSERT. Use `_write_lock` for all writes. Return True on success.
- `add_edge(source_id, target_id, relation, confidence, source_record) -> bool`: INSERT OR IGNORE (unique constraint handles dedup). Use `_write_lock`.
- `_quarantine_contradiction(node_id, existing_value, incoming_value, existing_confidence, incoming_confidence, source, record_id)`: INSERT into kg_contradictions with status='pending'.
- `get_node(node_id) -> dict | None`, `get_edges_from(node_id) -> list[dict]`, `get_edges_to(node_id) -> list[dict]`: Read queries (no lock needed, WAL mode allows concurrent reads).
- `count_nodes() -> int`, `count_edges() -> int`, `count_locked() -> int`, `count_pending_contradictions() -> int`: Aggregate queries for status reporting.

**knowledge/facts.py -- FactExtractor class:**
- `FactTriple` as a NamedTuple: subject (str), predicate (str), object_val (str), confidence (float).
- `FactExtractor` with domain-specific regex PATTERNS list following research example:
  - Health: "takes/prescribed/on [medication] for/daily/..." -> (health.medication.{normalized}, takes, medication_name, 0.75)
  - Schedule: "meeting/appointment/event at/on/with ..." -> (ops.schedule.{normalized}, has_event, event_desc, 0.65)
  - Preference: "prefers/likes/favorite ..." -> (preference.{normalized}, prefers, pref_value, 0.70)
  - Family: "son/daughter/wife/husband named ..." -> (family.member.{normalized}, family_relation, name, 0.80)
  - Location: "lives in/address is/located at ..." -> (ops.location.{normalized}, located_at, location, 0.70)
  - Finance: "salary/income/makes [amount]" -> (finance.income.{normalized}, earns, amount, 0.65)
- `extract(text, source, branch) -> list[FactTriple]`: Run all patterns, cap at 10 results, skip matches with object_val < 2 or > 100 chars.
- `_normalize(text) -> str`: Lowercase, replace whitespace with underscore, strip non-alphanumeric except underscores.

**memory/engine.py modifications:**
- In `_init_schema()`, after the existing schema, call a new method `_init_kg_schema()` that creates the three kg tables and bumps schema_version if not already at 2. This keeps the knowledge graph schema co-located with the memory schema but logically separated.
- IMPORTANT: The `_init_kg_schema()` method should be idempotent (CREATE IF NOT EXISTS).

**pyproject.toml:**

**memory/ingest.py -- Extend EnrichedIngestPipeline:**
- Add optional `knowledge_graph` parameter to `__init__` (KnowledgeGraph | None). Store as `self._kg`.
- After the classify step in the `ingest()` method (after records are stored in SQLite), if `self._kg` is not None:
  1. Extract facts from the content using `FactExtractor().extract(content, source, branch)`.
  2. For each FactTriple, call `self._kg.add_fact(triple.subject, triple.object_val, triple.confidence, source_record=record_id, node_type=triple.predicate)`.
  3. For each FactTriple, add an edge: `self._kg.add_edge("ingest:" + record_id, triple.subject, "extracted_from", confidence=triple.confidence, source_record=record_id)`.
- This is a SIDE EFFECT of ingestion -- fact extraction failure should NOT prevent the record from being stored. Wrap the fact extraction in try/except and log warnings on failure.
- Import FactExtractor lazily inside the ingest method to avoid circular imports.

**memory/__init__.py:**
- Add exports: KnowledgeGraph, FactExtractor, FactTriple from knowledge package.

**app.py -- Wire KnowledgeGraph into DI:**
- After creating the MemoryEngine instance (in the `if db_path.exists()` block), create a KnowledgeGraph instance: `kg = KnowledgeGraph(engine)`.
- Pass `knowledge_graph=kg` to EnrichedIngestPipeline constructor.
- This means the knowledge graph is automatically initialized whenever the SQLite engine is available, and fact extraction happens on every ingest.
