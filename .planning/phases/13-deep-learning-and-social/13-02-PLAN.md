    **CallStateReceiver** (`feature/social/CallStateReceiver.kt`): A `BroadcastReceiver` that listens for `TelephonyManager.ACTION_PHONE_STATE_CHANGED` broadcasts. Uses the EntryPointAccessors pattern (same as DoseAlarmReceiver/CallScreeningService) since BroadcastReceivers cannot use @AndroidEntryPoint directly.

    Define `@EntryPoint @InstallIn(SingletonComponent::class) interface CallStateEntryPoint` with:
    - `fun preCallCardManager(): PreCallCardManager`
    - `fun postCallLogger(): PostCallLogger`

    On receive:
    1. Extract phone state from `intent.getStringExtra(TelephonyManager.EXTRA_STATE)`.
    2. Extract incoming number from `intent.getStringExtra(TelephonyManager.EXTRA_INCOMING_NUMBER)` (may be null on Android 10+ -- fallback to empty).
    3. Track call state transitions using companion object fields:
       - `private var lastState = TelephonyManager.EXTRA_STATE_IDLE`
       - `private var callStartTime = 0L`
       - `private var currentNumber = ""`
    4. State handling:
       - **RINGING** (incoming call): Set currentNumber, call `preCallCardManager.showPreCallCard(number)` on a coroutine (use `CoroutineScope(Dispatchers.IO).launch { ... }`).
       - **OFFHOOK** (call answered or outgoing started): Record `callStartTime = System.currentTimeMillis()`. If previous state was IDLE (outgoing call), try to get the number from intent. Call `preCallCardManager.showPreCallCard(number)` if number available.
       - **IDLE** (call ended): If previous state was OFFHOOK (call just ended), calculate duration = `(System.currentTimeMillis() - callStartTime) / 1000`. Call `postCallLogger.promptForContext(currentNumber, duration, direction)` where direction is "incoming" if lastState transition was RINGING->OFFHOOK->IDLE, or "outgoing" if IDLE->OFFHOOK->IDLE. Reset tracking fields.
    5. Update `lastState` after processing.

    IMPORTANT: This receiver will be registered at runtime (in JarvisService) via `registerReceiver()`, NOT in the manifest. This gives us better lifecycle control and avoids needing the deprecated PROCESS_OUTGOING_CALLS permission.

    Actually -- register in AndroidManifest for `android.intent.action.PHONE_STATE` (this is the standard broadcast for call state changes, and the app already has READ_PHONE_STATE permission). Manifest registration ensures we catch calls even if the service hasn't started yet.
