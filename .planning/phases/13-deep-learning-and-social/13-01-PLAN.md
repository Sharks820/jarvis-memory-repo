    **PatternDetector** (`feature/habit/PatternDetector.kt`): Hilt `@Singleton` class injecting `HabitDao`, `ContextStateDao`, `CommuteDao`, and `@ApplicationContext context: Context`. Provides:

    - `suspend fun detectPatterns()` -- the main detection loop called periodically from JarvisService:
      1. **Time-based patterns**: Query `contextStateDao` for the last 14 days of context state entries. Group by day-of-week + hour. If the same context (e.g. "NORMAL" at gym location, or specific app usage) appears on the same day-of-week + hour window (within +/- 30 minutes) at least 3 times in 14 days, create or update a HabitPatternEntity with confidence = occurrences / 14.0 (capped at 1.0).
      2. **Location-based patterns**: Query `commuteDao` for learned locations. If the user visits a non-home, non-work location at consistent times (same day-of-week, similar hour), create a pattern like "You usually visit [location] at [time] on [day]."
      3. **Built-in pattern seeding**: Call `BuiltInNudges.ensureBuiltInPatterns(habitDao)` to create built-in nudge patterns if they don't already exist (water, screen time, sleep).
      4. For each detected pattern, check if a matching pattern already exists via `habitDao.findByTypeAndLabel()`. If so, call `habitDao.incrementOccurrence()` with recalculated confidence. If new, insert via `habitDao.insert()`.

    - `private fun calculateConfidence(occurrences: Int, totalDays: Int): Float` -- returns `(occurrences.toFloat() / totalDays.coerceAtLeast(1)).coerceIn(0f, 1f)`

    Pattern detection is intentionally simple and rule-based (no ML). It relies on existing data sources: ContextStateEntity timestamps, CommuteLocationEntity visit patterns, and time-of-day clustering.

    **NudgeEngine** (`feature/habit/NudgeEngine.kt`): Hilt `@Singleton` class injecting `HabitDao`, `NudgeLogDao`, `NudgeResponseTracker`, `NotificationChannelManager`, `BuiltInNudges`, and `@ApplicationContext context: Context`. Provides:

    - `suspend fun checkAndDeliver()` -- called periodically from JarvisService:
      1. Get current day-of-week (1=Mon..7=Sun using `java.time.LocalDate.now().dayOfWeek.value`) and current hour/minute.
      2. Query `habitDao.getActivePatterns()` (filters out suppressed and inactive).
      3. For each pattern, check if:
         a. Current day-of-week is in the pattern's triggerDays JSON array.
         b. Current time is within 15 minutes of the pattern's triggerHour:triggerMinute.
         c. No nudge has been delivered for this pattern today (check `nudgeLogDao.getLogsForDate(today)` for this patternId).
         d. `responseTracker.shouldSuppress(pattern.id)` returns false.
      4. If all checks pass, deliver the nudge:
         - Post a notification on the ROUTINE channel (`NotificationPriority.ROUTINE`).
         - Title: pattern.label
         - Body: pattern.description
         - Add action button "Done" that broadcasts `ACTION_NUDGE_ACTED` with extras `nudge_log_id`.
         - Add action button "Dismiss" that broadcasts `ACTION_NUDGE_DISMISSED` with extras `nudge_log_id`.
         - Notification ID: `(pattern.id + 20000).toInt()` to avoid collision.
         - Insert a NudgeLogEntity with `response = ""` (pending).
      5. Also check built-in nudges via `builtInNudges.getActiveBuiltIns()` and deliver those at their configured intervals.

    - `companion object` with:
      - `const val ACTION_NUDGE_ACTED = "com.jarvis.assistant.NUDGE_ACTED"`
      - `const val ACTION_NUDGE_DISMISSED = "com.jarvis.assistant.NUDGE_DISMISSED"`
      - `const val EXTRA_NUDGE_LOG_ID = "nudge_log_id"`

    **NudgeResponseTracker** (`feature/habit/NudgeResponseTracker.kt`): Hilt `@Singleton` class injecting `NudgeLogDao` and `HabitDao`. Provides:

    - `suspend fun shouldSuppress(patternId: Long): Boolean` -- queries the last 20 nudge logs for this pattern. If total >= 5 and the dismiss/expired rate >= 80% (i.e., fewer than 20% acted on), returns true. Also calls `habitDao.suppress(patternId)` to mark the pattern as suppressed so future queries skip it.
    - `suspend fun recordResponse(nudgeLogId: Long, response: String)` -- updates the NudgeLogEntity via `nudgeLogDao.updateResponse()`. If response is "acted", also increments the associated pattern's occurrence count (positive reinforcement).
    - `suspend fun expireStaleNudges()` -- calls `nudgeLogDao.expireOldNudges(System.currentTimeMillis() - 2 * 60 * 60 * 1000)` to mark unresponded nudges older than 2 hours as "expired". Run this periodically.
    - `suspend fun getResponseRate(patternId: Long): Float` -- returns acted / total ratio (0.0-1.0). Returns 1.0 if no data (assume engaged until proven otherwise).
