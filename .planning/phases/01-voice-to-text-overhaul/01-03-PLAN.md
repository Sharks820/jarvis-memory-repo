   ```python
   _keyterms_cache: list[str] | None = None

   def _load_keyterms() -> list[str]:
       """Load keyterms from personal_vocab.txt for Deepgram prompting."""
       global _keyterms_cache
       if _keyterms_cache is not None:
           return _keyterms_cache
       vocab_path = Path(__file__).parent / "data" / "personal_vocab.txt"
       if not vocab_path.exists():
           _keyterms_cache = []
           return _keyterms_cache
       try:
           lines = vocab_path.read_text(encoding="utf-8").strip().splitlines()
           _keyterms_cache = [line.strip() for line in lines if line.strip()]
           return _keyterms_cache
       except OSError:
           _keyterms_cache = []
           return _keyterms_cache
   ```

2. **`_try_deepgram()` function** following existing backend pattern:
   ```python
   def _try_deepgram(
       audio: np.ndarray | str, *, language: str, prompt: str = "",
       keyterms: list[str] | None = None,
   ) -> TranscriptionResult | None:
   ```

   Implementation details:
   - Check `DEEPGRAM_API_KEY` env var first -- return None immediately if not set (per research pitfall 4, avoids latency from failed connection)
   - Lazy import: `from deepgram import DeepgramClient` inside try/except ImportError -> return None
   - Create `DeepgramClient(api_key)` -- note: research says SDK v6 uses `client.listen.v1.media.transcribe_file()` but this needs validation. Try the documented API first. If SDK surface differs, fall back to direct httpx POST to `https://api.deepgram.com/v1/listen` as research suggests.
   - Convert numpy to WAV bytes using existing `_numpy_to_wav_bytes()` helper
   - Build options dict: `{"model": "nova-3", "language": language, "punctuate": True}`
   - If keyterms is None, auto-load from `_load_keyterms()`
   - Add keyterms to options: `options["keyterm"] = keyterms` (up to 500 tokens per research)
   - Extract transcript and per-word confidence from response
   - Return `TranscriptionResult(text=transcript, language=language, confidence=confidence, duration_seconds=elapsed, backend="deepgram-nova3")`
   - Wrap entire body in try/except Exception -> log warning, return None
   - Time with `time.monotonic()`

3. **SDK v6 API validation note**: The exact SDK v6 API for passing keyterms is marked as open question in research. The implementation should:
   - First try: `client.listen.rest.v("1").transcribe_file({"buffer": audio_bytes}, options)`
   - If that doesn't work, try: `client.listen.v1.media.transcribe_file(request=audio_bytes, options=options)` per research
   - If SDK is problematic, implement direct httpx fallback:
     ```python
     resp = httpx.post(
         "https://api.deepgram.com/v1/listen",
         headers={"Authorization": f"Token {api_key}", "Content-Type": "audio/wav"},
         params={"model": "nova-3", "language": language, "punctuate": "true", **{f"keyterm": kt for kt in keyterms}},
         content=audio_bytes,
     )
     ```
   - NOTE: httpx doesn't support duplicate query params easily. For multiple keyterms, build URL manually or use `params` as list of tuples: `params=[("model", "nova-3"), ("keyterm", "Jarvis"), ("keyterm", "Conner"), ...]`

4. **Do NOT modify `transcribe_smart()` yet** -- fallback chain rewrite is Plan 04.
