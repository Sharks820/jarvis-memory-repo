Rewrite `transcribe_smart()` in stt.py to use a 4-tier fallback chain:

1. **Define FALLBACK_CHAIN** as a module-level constant (after the backend functions):
   ```python
   FALLBACK_CHAIN = [
       ("parakeet", _try_parakeet),      # Best local: 6.05% WER
       ("deepgram", _try_deepgram),       # Best cloud: keyterm boosting
       ("groq", _try_groq),              # Existing cloud: free tier
       ("local", _try_local_emergency),   # Emergency: faster-whisper large-v3
   ]
   ```

2. **Create `_try_local_emergency()`**: A wrapper around `_try_local()` that forces model_size="large-v3" for the emergency fallback tier. Modify the `SpeechToText` default from "small.en" to use env var `JARVIS_STT_MODEL` (already exists) but change the hardcoded default to "large-v3". OR create a separate `_local_emergency_instance` with `SpeechToText(model_size="large-v3")`. The latter is safer -- keeps small.en backward compatible for anyone who set `JARVIS_STT_BACKEND=local`.

3. **Rewrite `transcribe_smart()` auto mode** (the `else` branch starting at line 580):
   - Replace the current groq-then-local logic with iteration over FALLBACK_CHAIN
   - For each `(name, try_fn)` in FALLBACK_CHAIN:
     - Call `try_fn(audio, language=language, prompt=prompt)` (pass keyterms for deepgram)
     - If result is not None and result.text.strip():
       - Log: `f"{name} STT: '{result.text[:60]}' in {result.duration_seconds:.2f}s (confidence: {result.confidence:.3f})"`
       - Log metric via `_log_stt_metric()`
       - If result.confidence >= CONFIDENCE_RETRY_THRESHOLD: break (good enough)
       - If result.confidence < CONFIDENCE_RETRY_THRESHOLD: store as `best_so_far`, continue to next backend
     - If result is None: log `f"{name} STT failed, trying next backend"`, continue
   - After loop: use `best_so_far` if no backend exceeded threshold, or the last successful result
   - If all backends failed: return empty TranscriptionResult with backend="none"

4. **Remove the old `_confidence_retry()` function**: The fallback chain IS the retry mechanism now. Each backend tries in order; low confidence falls through to next. No more single-retry logic.

5. **Handle `JARVIS_STT_BACKEND` forced modes**: Keep "groq", "local" forced modes working:
   - "groq": only try _try_groq (existing behavior)
   - "local": only try _try_local (existing behavior, still uses small.en or env override)
   - "parakeet": NEW -- force _try_parakeet only
   - "deepgram": NEW -- force _try_deepgram only
   - "auto" (default): use FALLBACK_CHAIN

6. **Post-processing**: Keep the existing post-processing block at the bottom of transcribe_smart() exactly as-is. It already handles `gateway` and `entity_list` parameters.

Add/update tests:

- **test_transcribe_smart_parakeet_primary**: Mock parakeet to succeed with high confidence. Verify deepgram/groq/local never called.

2. **Replace energy pre-filter** (lines 119-123):
   - Current: `rms = float(np.sqrt(np.mean(audio_int16.astype(np.float32) ** 2)) / 32767.0)` then `if rms < 0.005: continue`
   - New: Use float32 audio directly from sounddevice (before int16 conversion):
     ```python
     audio_float = audio_data[:, 0]  # mono channel, already float32 from sounddevice

         has_speech = vad.process_chunk(audio_float)  # handles 1280-sample chunk via sub-windowing
         if not has_speech:
             _was_silent = True
             continue
     else:
         # RMS fallback
         rms = float(np.sqrt(np.mean(audio_int16.astype(np.float32) ** 2)) / 32767.0)
         if rms < 0.005:
             _was_silent = True
             continue
     ```
