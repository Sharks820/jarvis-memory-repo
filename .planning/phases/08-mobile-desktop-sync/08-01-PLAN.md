# Plan 08-01: Changelog Table, Sync Engine, and Encrypted Transport

## Objective

Create the sync infrastructure: SQLite changelog table with triggers on records/kg_nodes/kg_edges, cursor-based incremental diff engine, Fernet-encrypted transport layer, and field-level conflict resolution.

## Requirements Covered

- SYNC-01: Changelog-based bidirectional sync between desktop and mobile
- SYNC-02: Only changes since last sync are transmitted (cursor-based diff)
- SYNC-03: Field-level conflict resolution with desktop as authoritative for ties
- SYNC-04: Sync payloads are encrypted in transit via Fernet

## Tasks

### Task 1: Create sync/changelog.py with changelog table and triggers

**Files to create:**
- `engine/src/jarvis_engine/sync/__init__.py` (NEW)
- `engine/src/jarvis_engine/sync/changelog.py` (NEW)

**`__init__.py` exports:** install_changelog_triggers, compute_diff, get_sync_cursor, update_sync_cursor

**Schema (in changelog.py):**
- `_sync_changelog` table: changelog_id (INTEGER PK AUTOINCREMENT), table_name (TEXT), row_id (TEXT), operation (TEXT: INSERT/UPDATE/DELETE), fields_changed (TEXT JSON array), old_values (TEXT JSON), new_values (TEXT JSON), device_id (TEXT DEFAULT 'desktop'), ts (TEXT DEFAULT datetime('now')), __version (INTEGER)
- `_sync_cursor` table: device_id (TEXT), table_name (TEXT), last_version (INTEGER DEFAULT 0), last_sync_ts (TEXT DEFAULT ''), PRIMARY KEY (device_id, table_name)
- Indexes: idx_changelog_version (table_name, __version), idx_changelog_device (device_id)

**Constants:**

**Functions:**
1. `install_changelog_triggers(db, device_id='desktop') -> None` -- Idempotent. Creates changelog + cursor tables, then generates AFTER INSERT/UPDATE/DELETE triggers for each tracked table. Version computed as MAX(__version)+1 per table. UPDATE triggers use WHEN clause checking if any tracked field actually changed (exclude access_count/last_accessed from WHEN for records to reduce noise, but still log them if other fields changed).
2. `compute_diff(db, table_name, since_version, limit=1000) -> list[dict]` -- SELECT from _sync_changelog WHERE table_name=? AND __version > ? ORDER BY __version ASC LIMIT ?
3. `get_sync_cursor(db, device_id, table_name) -> int` -- Returns last_version or 0
4. `update_sync_cursor(db, device_id, table_name, version, write_lock) -> None` -- UPSERT into _sync_cursor
5. `compact_changelog(db, write_lock, retention_days=7) -> int` -- DELETE entries older than retention_days AND where all devices have synced past that version. Returns count deleted.

### Task 2: Create sync/engine.py with diff computation and conflict resolution

**Files to create:**
- `engine/src/jarvis_engine/sync/engine.py` (NEW)

**SyncEngine class:**
- Constructor: `db, write_lock, device_id='desktop'`
- `compute_outgoing(target_device_id, limit=500) -> dict` -- For each tracked table, compute_diff since target's cursor. Return `{"changes": {table: [entries]}, "cursors": {table: max_version}}`
- `apply_incoming(changes: dict, source_device_id: str) -> dict` -- Apply remote changes within a single transaction. For each entry: check if local has a conflicting change (same row_id modified locally since last sync from that device). If conflict, use `_resolve_conflict()`. Otherwise apply directly. Return `{"applied": N, "conflicts_resolved": N, "errors": []}`
- `_resolve_conflict(local_entry, remote_entry, desktop_is_local) -> dict` -- Field-level merge. Desktop wins ties. DELETE always wins over UPDATE.
- `_apply_single_change(table_name, entry) -> None` -- Execute INSERT/UPDATE/DELETE SQL based on entry operation. For INSERT: INSERT OR IGNORE (idempotent). For UPDATE: update only fields in fields_changed. For DELETE: DELETE by row_id.
- `sync_status() -> dict` -- Return cursors for all devices and tables, plus total changelog size

### Task 3: Create sync/transport.py with Fernet encryption

**Files to create:**
- `engine/src/jarvis_engine/sync/transport.py` (NEW)

**Functions:**
1. `derive_sync_key(signing_key: str, salt: bytes) -> bytes` -- PBKDF2HMAC(SHA256, 32 bytes, 480_000 iterations) then base64 urlsafe encode for Fernet
2. `get_or_create_salt(salt_path: Path) -> bytes` -- If salt file exists, read and return. Otherwise generate 16 random bytes via os.urandom, write to file, return. Set file permissions 0o600.
3. `encrypt_sync_payload(payload: dict, fernet_key: bytes) -> bytes` -- json.dumps -> zlib.compress(level=6) -> Fernet.encrypt
4. `decrypt_sync_payload(token: bytes, fernet_key: bytes) -> dict` -- Fernet.decrypt -> zlib.decompress -> json.loads
5. `SyncTransport` class:
   - Constructor: `signing_key: str, salt_path: Path`
   - Lazy derives sync key on first use
   - `encrypt(payload: dict) -> bytes`
   - `decrypt(token: bytes) -> dict`
