# Plan 09-01: Proactive Trigger Engine, Notifications, and Wake Word Detection

## Objective

Build a proactive assistance system that surfaces relevant info (bill reminders, medication alerts, meeting prep) without being asked, plus replace the PowerShell-based hotword detection with openwakeword "hey_jarvis" ML model for hands-free voice activation.

## Requirements Covered

- CONN-05: Proactive assistance surfaces relevant info without being asked
- VOICE-05: Wake word detection ("Hey Jarvis") enables hands-free voice activation

## Tasks

### Task 1: Create proactive/triggers.py with time-aware trigger rules

**Files to create:**
- `engine/src/jarvis_engine/proactive/__init__.py` (NEW)
- `engine/src/jarvis_engine/proactive/triggers.py` (NEW)

**Dataclasses:**
1. `TriggerRule`: rule_id (str), description (str), check_fn (Callable[[dict], list[str]]), cooldown_minutes (int = 60)
2. `TriggerAlert`: rule_id (str), message (str), priority (str = "normal"), timestamp (str)

**Built-in trigger check functions (module-level):**
1. `check_medication_reminders(snapshot_data: dict) -> list[str]` -- Check medications list for items with due_time within 30 minutes of now. Parse HH:MM format.
2. `check_bill_due_alerts(snapshot_data: dict) -> list[str]` -- Check bills for status=due or overdue.
3. `check_calendar_prep(snapshot_data: dict) -> list[str]` -- Check calendar_events for prep_needed=true within next 2 hours.
4. `check_urgent_tasks(snapshot_data: dict) -> list[str]` -- Check tasks for priority=high or urgent.

**DEFAULT_TRIGGER_RULES list:**
- medication_reminder (cooldown=30min)
- bill_due_alert (cooldown=360min)
- calendar_prep (cooldown=120min)
- urgent_task_alert (cooldown=180min)

### Task 2: Create proactive/notifications.py with Windows toast delivery

**Files to create:**
- `engine/src/jarvis_engine/proactive/notifications.py` (NEW)

**Notifier class:**
- Constructor: `app_name: str = "Jarvis"`
- `send(title: str, message: str, priority: str = "normal") -> bool` -- Try winotify Notification. If winotify unavailable, fall back to logger.info. Return True on success.

- Lazy import of winotify (graceful degradation if not installed)

### Task 3: Create ProactiveEngine class

**Files to modify:**
- `engine/src/jarvis_engine/proactive/__init__.py` (MODIFY -- add ProactiveEngine here)

**ProactiveEngine class:**
- Constructor: `rules: list[TriggerRule], notifier: Notifier`
- `_last_fired: dict[str, datetime]` -- tracks cooldowns
- `evaluate(snapshot_data: dict) -> list[TriggerAlert]` -- For each rule, check cooldown, call check_fn, create TriggerAlert for each message, send via notifier. Return list of alerts fired.
- `reset_cooldowns() -> None` -- Clear all cooldown state
- Respects cooldown_minutes per rule

**`__init__.py` exports:** ProactiveEngine, TriggerRule, TriggerAlert, Notifier, DEFAULT_TRIGGER_RULES

### Task 4: Create wakeword.py with openwakeword integration

**Files to create:**
- `engine/src/jarvis_engine/wakeword.py` (NEW)

**WakeWordDetector class:**
- Constructor: `threshold: float = 0.5, model_name: str = "hey_jarvis"`
- `_model = None` (lazy-loaded)
- `_load_model() -> None` -- lazy import openwakeword.model.Model, load with inference_framework="onnx"
- `start(on_detected: Callable, stop_event: threading.Event, mic_lock: threading.Lock | None = None) -> None` -- Main detection loop using sounddevice (already installed, avoid pyaudio dependency). Open InputStream at 16kHz mono, chunk=1280 frames. Convert float32 to int16 for openwakeword. On detection (score > threshold), acquire mic_lock if provided (prevents conflict with STT), call on_detected, reset model.
- `stop() -> None` -- Set internal stop event
- Graceful degradation: if openwakeword not installed, log warning and return immediately

**Thread-safe mic sharing with STT:**
- Accept optional mic_lock from STT module
- Release mic before calling on_detected callback
- Re-acquire mic after callback returns
