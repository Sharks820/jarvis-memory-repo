# Plan 09-02: Progressive Cost Reduction and Adversarial Self-Testing

## Objective

Extend CostTracker with local-vs-cloud query ratio tracking, add cost reduction trend snapshots, and build adversarial self-testing that periodically quizzes Jarvis on retained knowledge with alerting on score drops.

## Requirements Covered

- INTL-05: Progressive cost reduction tracked as local knowledge grows
- GROW-03: Adversarial self-testing periodically quizzes Jarvis on retained knowledge
- GROW-04: Percentage of queries answered locally vs cloud tracked over time

## Tasks

### Task 1: Extend CostTracker with local_vs_cloud_summary

**Files to modify:**
- `engine/src/jarvis_engine/gateway/costs.py` (MODIFY)

**New method on CostTracker:**
- `local_vs_cloud_summary(self, days: int = 30) -> dict` -- Query query_costs table grouped by CASE WHEN provider='ollama' THEN 'local' ELSE 'cloud' END. Return dict with: period_days, local_count, cloud_count, total_count, local_pct (rounded to 1 decimal), cloud_cost_usd.

### Task 2: Create cost reduction trend tracking

**Files to create:**
- `engine/src/jarvis_engine/proactive/cost_tracking.py` (NEW)

**Functions:**
1. `cost_reduction_snapshot(cost_tracker, history_path: Path) -> dict` -- Compute 7d and 30d summaries via local_vs_cloud_summary, write JSONL line to history_path, return snapshot dict with: date, 7d_local_pct, 30d_local_pct, 7d_cloud_cost_usd, 30d_cloud_cost_usd, 7d_total_queries, 30d_total_queries.
2. `load_cost_history(history_path: Path, limit: int = 90) -> list[dict]` -- Read last N lines from JSONL file, return list of snapshot dicts.
3. `cost_reduction_trend(history: list[dict]) -> dict` -- Compute trend: compare first and last entry's 30d_local_pct. Return dict with: first_date, last_date, first_local_pct, last_local_pct, change_pct, trend ("improving", "stable", "declining").

### Task 3: Build adversarial self-testing framework

**Files to create:**
- `engine/src/jarvis_engine/proactive/self_test.py` (NEW)

**AdversarialSelfTest class:**
- Constructor: `engine, embed_service, notifier=None, score_threshold: float = 0.5`
- `run_memory_quiz(tasks: list[MemoryRecallTask] | None = None) -> dict` -- If tasks is None, use DEFAULT_MEMORY_TASKS from growth_tracker. Call run_memory_eval(tasks, engine, embed_service). Compute average score. If score < threshold and notifier, send alert. Return dict with: tasks_run, average_score, below_threshold, per_task_scores, timestamp.
- `save_quiz_result(result: dict, history_path: Path) -> None` -- Append to JSONL history file.
- `check_regression(history_path: Path, window: int = 5) -> dict` -- Load last `window` quiz results. If latest score < average of previous scores * 0.8, flag regression. Return dict with: regression_detected, current_score, baseline_score, drop_pct.

### Task 4: Wire cost tracking and self-test commands to Command Bus

**Files to create/modify:**
- `engine/src/jarvis_engine/commands/proactive_commands.py` (MODIFY -- add new commands)
- `engine/src/jarvis_engine/handlers/proactive_handlers.py` (MODIFY -- add new handlers)
- `engine/src/jarvis_engine/app.py` (MODIFY)

**New Commands:**
1. `CostReductionCommand(frozen)`: days (int = 30)
2. `CostReductionResult`: local_pct (float = 0.0), cloud_cost_usd (float = 0.0), trend (str = ""), message (str = "")
3. `SelfTestCommand(frozen)`: score_threshold (float = 0.5)
4. `SelfTestResult`: average_score (float = 0.0), tasks_run (int = 0), regression_detected (bool = False), message (str = "")

**New Handlers:**
1. `CostReductionHandler(root, cost_tracker=None)` -- Call local_vs_cloud_summary, take snapshot, compute trend
2. `SelfTestHandler(root, engine=None, embed_service=None)` -- Run AdversarialSelfTest.run_memory_quiz, save result, check regression

**app.py changes:**
- Register CostReductionCommand and SelfTestCommand
- Pass cost_tracker, engine, embed_service to handlers
- Graceful degradation if cost_tracker or engine is None

### Task 5: Add CLI subcommands

**Files to modify:**
- `engine/src/jarvis_engine/main.py` (MODIFY)

**New subcommands:**
