    **MedicationScheduler** (`feature/prescription/MedicationScheduler.kt`): Hilt `@Singleton` class injecting `@ApplicationContext context: Context` and `MedicationDao`. Provides:
    - `suspend fun scheduleAllAlarms()` -- queries all active medications, for each scheduled time today (and tomorrow if past today's times), creates an exact alarm via `AlarmManager.setExactAndAllowWhileIdle()`. The PendingIntent targets `DoseAlarmReceiver` with extras: `medication_id` (Long), `medication_name` (String), `dosage` (String), `scheduled_time` (String HH:mm). Uses request code = `(medicationId * 100 + timeIndex).toInt()` for uniqueness. Intent action = `"com.jarvis.assistant.DOSE_ALARM"`.
    - `suspend fun cancelAllAlarms()` -- cancels all pending alarms for active medications using matching PendingIntents.
    - `suspend fun rescheduleForMedication(medicationId: Long)` -- cancels and re-creates alarms for a single medication (call after edit).
    - `private fun getNextAlarmTimeMs(hourMinute: String): Long` -- parses "HH:mm", returns the next occurrence as epoch millis. If the time has already passed today, returns tomorrow's occurrence.

    IMPORTANT: Use `AlarmManager.setExactAndAllowWhileIdle()` (not `setExact()` or `setRepeating()`) because it fires even in Doze mode. Use `PendingIntent.FLAG_IMMUTABLE or PendingIntent.FLAG_UPDATE_CURRENT`. Check `AlarmManager.canScheduleExactAlarms()` before scheduling (Android 12+ requires SCHEDULE_EXACT_ALARM permission).

    **DoseAlarmReceiver** (`feature/prescription/DoseAlarmReceiver.kt`): Extends `BroadcastReceiver`. On receive:
    1. Extract `medication_name`, `dosage`, `medication_id`, `scheduled_time` from intent extras.
    2. Build a notification on the `jarvis_urgent` channel (URGENT priority -- bypasses DND):
       - Title: "Medication Reminder"
       - Body: "Time to take {name} ({dosage})"
       - Add action button "Taken" that broadcasts to a `DoseTakenReceiver` (inner class or separate) which logs the dose via `MedicationLogDao.insert()` and calls `MedicationDao.decrementPills()`.
       - Add action button "Skip" that logs with status "skipped".
       - Use notification ID = `medication_id.toInt() + 10000` to avoid collision with other notifications.
    3. Also schedule the next occurrence of this alarm (for tomorrow at the same time) via `MedicationScheduler`.

    NOTE: Because `DoseAlarmReceiver` is a `BroadcastReceiver` (not `@AndroidEntryPoint` compatible for Hilt directly), use `EntryPointAccessors.fromApplication()` pattern (same as used for `JarvisNotificationListenerService` in Phase 11) to get DAOs. Define an `@EntryPoint` interface `DoseAlarmEntryPoint` with `fun medicationDao(): MedicationDao`, `fun medicationLogDao(): MedicationLogDao`, and `fun medicationScheduler(): MedicationScheduler`.

    **RefillTracker** (`feature/prescription/RefillTracker.kt`): Hilt `@Singleton` class injecting `MedicationDao`, `NotificationChannelManager`, and `@ApplicationContext context: Context`. Provides:
    - `suspend fun checkRefills()` -- queries all active medications. For each, calculates `daysRemaining = pillsRemaining / dosesPerDay` (where dosesPerDay is derived from scheduledTimes count). If `daysRemaining <= medication.refillReminderDays` and a refill reminder hasn't been shown today (track in SharedPreferences with key `refill_last_reminded_{id}`), posts an IMPORTANT priority notification: "Refill Reminder: {name} -- {pillsRemaining} pills remaining (~{daysRemaining} days)".
    - `suspend fun syncToDesktop(apiClient: JarvisApiClient)` -- sends medication data to desktop brain via `/command` endpoint: `"Jarvis, remember medication: {name} {dosage} {frequency}, {pillsRemaining} pills remaining"`. Called after medication changes.

    **MedicationVoiceHandler** (`feature/prescription/MedicationVoiceHandler.kt`): Hilt `@Singleton` class injecting `MedicationLogDao` and `MedicationDao`. Provides:
    - `suspend fun handleQuery(query: String): String?` -- checks if query matches medication-related patterns (regex: `(?i)(did I take|have I taken|medication|meds|pills|medicine).*(?:today|morning|evening|night|afternoon)?`). If match:
      - Get today's date as YYYY-MM-DD string.
      - Query `medicationLogDao.getLogsForDate(today)`.
      - Query `medicationDao.getActiveMedications()` to know what was scheduled.
      - Build a natural language response:
        - If all scheduled doses taken: "Yes sir, you've taken all your medications today. {list of taken meds with times}."
        - If some taken, some pending: "You've taken {taken list}, but you still need to take {pending list}."
        - If none taken: "You haven't logged any doses yet today. You're scheduled for {list}."
      - Returns null if the query doesn't match medication patterns (so callers can fall through to other handlers).
    - `fun matchesMedicationQuery(query: String): Boolean` -- returns true if query text matches medication patterns. Used by VoiceEngine to route queries.

    - Add permission: `<uses-permission android:name="android.permission.USE_EXACT_ALARM" />` (Android 14+ alternative)
    - Register the receiver inside `<application>`:
