    NOTE on search: Room's LIKE operator is case-insensitive by default for ASCII in SQLite. For the initial implementation, SQL LIKE search is sufficient. Room does not natively support FTS (Full-Text Search) without an FTS4 virtual table, but since the database uses SQLCipher, FTS4 support may vary. Use LIKE '%query%' for now -- it works well for document-sized text. If performance becomes an issue with thousands of documents, an FTS4 shadow table can be added later.

    **DocumentScanner** (`feature/documents/DocumentScanner.kt`): Hilt `@Singleton` class injecting `@ApplicationContext context: Context`, `DocumentDao`, and `DocumentCategorizer`. Provides:
    - `suspend fun scanAndExtract(imageUri: Uri): ScannedDocumentEntity` -- the main pipeline:
      1. Load image from Uri into `InputImage.fromFilePath(context, imageUri)` (ML Kit API).
      2. Run `TextRecognition.getClient(TextRecognizerOptions.DEFAULT_OPTIONS).process(inputImage)` to get `Text` result.
      3. Extract full text: `text.textBlocks.joinToString("\n") { it.text }`.
      4. Calculate average confidence from text blocks (ML Kit provides confidence per block in newer versions; if unavailable, default to 0.8f).
      5. Save the original image to app-internal storage: `context.filesDir/documents/{timestamp}.jpg`. Use `context.contentResolver.openInputStream(imageUri)` to copy. Do NOT store the image as a BLOB in Room -- file system is more efficient for large binaries.
      6. Generate a thumbnail (compressed, 200x200 max) and save to `context.filesDir/documents/thumbs/{timestamp}.jpg`. Use `Bitmap.createScaledBitmap()` and compress with JPEG quality 60.
      7. Auto-generate title from first line of OCR text (truncated to 50 chars) or "Scan {date}" if OCR is empty.
      8. Compute SHA-256 hash of ocrText as contentHash.
      9. Call `documentCategorizer.categorize(ocrText)` to determine category.
      10. Create and insert ScannedDocumentEntity into Room via DocumentDao.
      11. Return the created entity.
    - `suspend fun deleteDocument(doc: ScannedDocumentEntity)` -- deletes the entity from Room AND deletes the image and thumbnail files from disk.

    **DocumentCategorizer** (`feature/documents/DocumentCategorizer.kt`): Hilt `@Singleton` class. Provides:

      - "insurance" if text contains: "insurance", "policy", "premium", "deductible", "coverage", "claim", "beneficiary", "insured"
      - "other" as default fallback
    - Uses case-insensitive matching. If multiple categories match, use priority order: id > medical > insurance > warranty > receipt > other (IDs and medical are most important to categorize correctly).

    **DocumentSearchEngine** (`feature/documents/DocumentSearchEngine.kt`): Hilt `@Singleton` class injecting `DocumentDao`. Provides:
    - `suspend fun search(query: String, category: String? = null): List<ScannedDocumentEntity>` -- parses the natural language query to extract search terms and optional date/category filters:
      1. Extract date hints: regex for month names ("January", "February", etc.) and year patterns ("2024", "2025", "2026"). If found, filter results by createdAt date range.
      2. Extract category hints: if query contains "receipt", "warranty", "ID", "medical", "insurance", map to category filter.
      3. Extract merchant/content terms: remove date and category words, remaining words are content search terms.
      4. Execute `documentDao.searchByContent(searchTerms)` or `documentDao.searchByContentAndCategory(searchTerms, category)`.
      5. If date hints were found, post-filter results by createdAt timestamp falling within the month/year.
      6. Return sorted results (most recent first).
    - Example: "find my Best Buy receipt from January" -> search terms = "Best Buy", category = "receipt", date filter = January of current year.

    Implementation approach for CameraX in Compose:
    ```kotlin
    @Composable
    fun CameraPreview(
        onImageCaptured: (Uri) -> Unit,
        modifier: Modifier = Modifier,
    ) {
        val context = LocalContext.current
        val lifecycleOwner = LocalLifecycleOwner.current
        val cameraProviderFuture = remember { ProcessCameraProvider.getInstance(context) }

        AndroidView(
            factory = { ctx ->
                PreviewView(ctx).also { previewView ->
                    cameraProviderFuture.addListener({
                        val cameraProvider = cameraProviderFuture.get()
                        val preview = Preview.Builder().build().also {
                            it.setSurfaceProvider(previewView.surfaceProvider)
                        }
                        val imageCapture = ImageCapture.Builder()
                            .setCaptureMode(ImageCapture.CAPTURE_MODE_MINIMIZE_LATENCY)
                            .build()
                        // Store imageCapture reference for the capture button
                        cameraProvider.unbindAll()
                        cameraProvider.bindToLifecycle(
                            lifecycleOwner, CameraSelector.DEFAULT_BACK_CAMERA,
                            preview, imageCapture,
                        )
                    }, ContextCompat.getMainExecutor(ctx))
                }
            },
            modifier = modifier,
        )
    }
    ```
