# Plan 07-01: Continuous Learning Engine with Automatic Knowledge Extraction

## Objective

Build a ConversationLearningEngine that extracts knowledge from every interaction and permanently retains it, plus add temporal metadata to KG facts and cross-branch edge creation for cross-domain reasoning.

## Requirements Covered

- GROW-01: Continuous learning engine extracts knowledge from every interaction
- KNOW-05: Cross-branch fact relationships enable cross-domain reasoning queries
- KNOW-06: Temporal metadata on facts distinguishes permanent from time-sensitive

## Tasks

### Task 1: Create learning module with ConversationLearningEngine

**Files to create:**
- `engine/src/jarvis_engine/learning/__init__.py` (NEW)
- `engine/src/jarvis_engine/learning/engine.py` (NEW)

**ConversationLearningEngine class:**
- Constructor: `pipeline: EnrichedIngestPipeline | None, kg: KnowledgeGraph | None`
- `learn_from_interaction(user_message: str, assistant_response: str, task_id: str = "") -> dict`
  - Filter with `_is_knowledge_bearing()` -- skip texts <50 chars, skip greetings/commands
  - Ingest user messages with source="conversation:user", kind="episodic", tags=["conversation","user"]
  - Ingest assistant responses with source="conversation:assistant", kind="semantic", tags=["conversation","assistant"]
  - Return `{"records_created": N}`
- `_is_knowledge_bearing(text: str) -> bool`
  - Returns False if text is None/empty or <50 chars
  - Returns False if text starts with greeting prefix AND <100 chars
  - Skip prefixes: "jarvis ", "hey ", "ok ", "thanks", "thank you", "goodbye"
- Graceful degradation: if pipeline is None, return `{"records_created": 0, "error": "no pipeline"}`

**`__init__.py` exports:** ConversationLearningEngine

### Task 2: Add temporal metadata to KG facts

**Files to create:**
- `engine/src/jarvis_engine/learning/temporal.py` (NEW)

**Schema migration function:**
- `migrate_temporal_metadata(db, write_lock)` -- idempotent ALTER TABLE adds temporal_type (TEXT DEFAULT 'unknown') and expires_at (TEXT DEFAULT NULL) to kg_nodes, with indexes

**Classification function:**
- `classify_temporal(node_id: str, label: str) -> tuple[str, str | None]`
  - Permanent prefixes: family.member, preference, ops.location, finance.income
  - Time-sensitive prefixes: ops.schedule, health.medication
  - Temporal date patterns via regex: expires/due/until/today/tomorrow + ISO dates
  - Returns (temporal_type, expires_at_iso_or_none)

**Expiration flagging:**
- `flag_expired_facts(kg) -> int` -- UPDATE kg_nodes SET temporal_type='expired' WHERE expires_at < now AND temporal_type != 'expired'

### Task 3: Add cross-branch reasoning

**Files to create:**
- `engine/src/jarvis_engine/learning/cross_branch.py` (NEW)

**Cross-branch query function:**
- `cross_branch_query(query, engine, kg, embed_service, k=10) -> dict`
  - Embed query via embed_service.embed(query, prefix="search_query")
  - Search via engine.search_vec(embedding, limit=k*2)
  - Get graph via kg.to_networkx()
  - For each result, find KG neighbors in other branches via node_id prefix
  - Return `{"direct_results": [...], "cross_branch_connections": [...], "branches_involved": [...]}`

**Cross-branch edge creation (at ingest time):**
- `create_cross_branch_edges(kg, new_fact_id, record_id) -> int`
  - Get node label, extract branch from node_id prefix
  - Search kg_nodes for label keyword overlap in other branches (LIKE queries, capped at 5 keywords, 10 results each)
  - Create 'cross_branch_related' edges at confidence=0.4
  - Return count of edges created

**Commands:**
1. `LearnInteractionCommand(frozen)`: user_message, assistant_response, task_id=""
2. `LearnInteractionResult`: records_created=0, message=""
3. `CrossBranchQueryCommand(frozen)`: query, k=10
4. `CrossBranchQueryResult`: direct_results=[], cross_branch_connections=[], branches_involved=[], message=""
5. `FlagExpiredFactsCommand(frozen)`: (empty)
6. `FlagExpiredFactsResult`: expired_count=0, message=""

**Handlers:**
1. `LearnInteractionHandler(root, learning_engine=None)` -- delegates to ConversationLearningEngine
2. `CrossBranchQueryHandler(root, engine=None, kg=None, embed_service=None)` -- delegates to cross_branch_query
3. `FlagExpiredFactsHandler(root, kg=None)` -- delegates to flag_expired_facts

**Bus registration in app.py:**
- Register all three in try/except block for graceful degradation
- Create ConversationLearningEngine with pipeline and kg
