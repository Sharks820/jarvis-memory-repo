<objective>
Build the Unity-domain prompting layer that injects Unity 6.3 API knowledge from the KG into LLM system prompts, and a pre-compilation API validator that catches wrong API usage before triggering compilation.

Purpose: Without domain-specific prompting, qwen3.5 hallucinates Unity 2020-2022 APIs that fail on Unity 6.3. The KG was seeded in Phase 20 with API references and breaking changes -- this plan makes that knowledge usable during code generation.

Output: UnityPromptBuilder (builds system prompts with KG facts), ApiValidator (validates generated C# against KG before compile).
</objective>

  <name>Task 2: ApiValidator -- pre-compilation C# validation against KG</name>
  <files>engine/src/jarvis_engine/agent/codegen/api_validator.py, engine/tests/test_codegen_api_validator.py</files>
  <behavior>
    - validate_csharp_against_kg(code, kg) returns ValidationResult with warnings: list[str] and suggestions: list[str]
    - Code using "UnityEngine.Experimental.Rendering" triggers a breaking change warning with the KG-sourced alternative
    - Code using "[SerializeField]" on a property (detected via regex: `\[SerializeField\]\s*public\s+\w+\s+\w+\s*\{`) triggers a warning suggesting [field: SerializeField]
    - Code referencing a class/method not found in KG API facts gets a "possibly incorrect API" warning (not a hard block -- KG coverage is incomplete)
    - When compile error CS0117 or CS0619 is provided, query_kg_for_alternative(error_msg, kg) returns the KG-sourced correct API if available
    - Empty code returns no warnings
  </behavior>
  <action>
    Create `engine/src/jarvis_engine/agent/codegen/api_validator.py` with:

    1. `ValidationResult` dataclass with `warnings: list[str]`, `suggestions: list[str]`.

    2. `ApiValidator` class accepting `KnowledgeGraph`:
       a. `validate(code: str) -> ValidationResult`:
          - Regex scan for `UnityEngine.Experimental.*` usage -- if found, query KG for the non-Experimental equivalent.
          - Regex scan for `[SerializeField]` on property declarations -- warn about `[field: SerializeField]`.
          - Extract `using` directives and major class references from the code. Query KG to check if they exist as unity_api facts. If a referenced type has no KG match, add a soft warning ("possibly incorrect for Unity 6.3").
          - Return ValidationResult with all collected warnings and suggestions.
       b. `query_alternative(error_code: str, error_msg: str) -> str | None`:
          - For CS0117/CS0619 errors, extract the type/member name from the error message.
          - Query KG with those keywords filtered to unity_api and unity_breaking node types.
          - If a match is found, return a suggestion string like "Use X instead of Y (per Unity 6.3 breaking changes)".
          - Return None if no alternative found.

    3. Module-level convenience: `validate_csharp_against_kg(code: str, kg: KnowledgeGraph) -> ValidationResult`.

    Use compiled regex patterns for detection. TYPE_CHECKING for KG import.
  </action>
