<task type="auto" tdd="true">
  <name>Task 2: CompileFixLoop -- autonomous compile-test-fix orchestrator</name>
  <files>engine/src/jarvis_engine/agent/codegen/compile_fix_loop.py, engine/tests/test_codegen_compile_fix_loop.py</files>
  <behavior>
    - run(script_path, script_content) returns CompileFixResult with success: bool, final_code: str, iterations: int, errors: list[str]
    - On clean compilation, returns success=True after 1 iteration
    - On CS0117 error, queries ApiValidator.query_alternative() and includes the suggestion in the fix prompt
    - On 5 consecutive failures, returns success=False with all errors collected
    - Play-mode entry acquires VRAMCoordinator.acquire_playmode() and releases on exit (even on error)
    - If NUnit tests fail, the error is fed back into the fix loop
    - Each fix iteration uses the LLM with the error context and ApiValidator suggestions to generate corrected code
  </behavior>
  <action>
    Create `engine/src/jarvis_engine/agent/codegen/compile_fix_loop.py` with:

    1. `CompileFixResult` dataclass: `success: bool`, `final_code: str`, `iterations: int`, `errors: list[str]`, `warnings: list[str]`.

    2. `CompileFixLoop` class:
       a. `__init__(self, unity_tool: UnityTool, gateway: ModelGateway, validator: ApiValidator, coordinator: VRAMCoordinator, prompt_builder: UnityPromptBuilder, max_retries: int = 5)`.
       b. `async run(self, script_path: str, script_content: str, test_path: str | None = None, test_content: str | None = None) -> CompileFixResult`:
          Main loop (up to max_retries):
          1. Pre-validate code with `validator.validate(current_code)` -- collect warnings.
          2. Write script via `unity_tool.write_script(script_path, current_code)`.
          3. If test_path and test_content provided, write test file too via `unity_tool.write_script(test_path, test_content)`.
          4. Trigger compilation via `unity_tool.compile()`.
          5. Parse compile result -- if errors:
             - For CS0117/CS0619: call `validator.query_alternative(error_code, error_msg)` to get KG suggestion.
             - Build a fix prompt with: current code, error messages, KG suggestions, breaking change warnings.
             - Call `gateway.complete()` with fix prompt to get corrected code.
             - Extract code from LLM response (strip markdown fences).
             - Set current_code = corrected code, continue loop.
          6. If compilation succeeds and test_path was provided:
             - Call `unity_tool.call("RunTests", {"testFilter": test_class_name})` to run NUnit tests.
             - If tests fail: feed test errors back into fix loop (same as compile errors).
          7. If compilation succeeds (and tests pass or no tests):
             - Acquire `coordinator.acquire_playmode()`.
             - Enter play mode via `unity_tool.call("EnterPlayMode", {})`.
             - Wait briefly, exit play mode via `unity_tool.call("ExitPlayMode", {})`.
             - Release `coordinator.release_playmode()` in a finally block.
             - Return CompileFixResult(success=True, ...).
          8. After max_retries, return CompileFixResult(success=False, ...).

    The fix prompt should use `prompt_builder.build_unity_system_prompt()` as the system message and include the specific errors and suggestions as user context. Use `_strip_code_fences()` (same logic as TaskPlanner) to clean LLM responses.

    VRAM coordinator release MUST be in a try/finally to prevent leaks. Play-mode is optional -- if unity_tool is not connected, skip play-mode gracefully.
  </action>
