  <action>
    Create `unity/com.jarvis.editor-bridge/Editor/JarvisPanel.cs` implementing an EditorWindow:

    1. **Menu registration:** `[MenuItem("Window/Jarvis/Agent Panel")]` static method `ShowWindow()` that opens or focuses the panel via `GetWindow<JarvisPanel>("Jarvis Agent")`.

    2. **State fields:**
       - `string _currentStep = "Idle"` -- name of the current agent step
       - `string _currentStatus = "idle"` -- status string (idle/running/waiting_approval/done/failed)
       - `List<string> _logEntries = new List<string>()` -- recent log lines (cap at 50, FIFO)
       - `Vector2 _logScrollPos` -- scroll position for log area
       - `bool _approvalPending = false` -- whether an approval request is active
       - `string _approvalTaskId = ""` -- task ID for pending approval
       - `string _approvalDescription = ""` -- what the agent wants to do

    3. **Message handling:**
       - `OnEnable()`: Register as a listener on JarvisEditorBridge for incoming messages. Use a static event `JarvisEditorBridge.OnAgentMessage += HandleAgentMessage` pattern. If JarvisEditorBridge does not yet expose this event, add a `public static event Action<string> OnAgentMessage;` to JarvisEditorBridge.cs and invoke it from the message dispatch when method starts with "agent_".
       - `OnDisable()`: Unregister the listener.
       - `HandleAgentMessage(string jsonMessage)`:
         * Parse JSON to extract `method` and `params`.
         * `agent_step_start`: Update `_currentStep` and `_currentStatus = "running"`.
         * `agent_step_done`: Update `_currentStatus`, add to `_logEntries`.
         * `agent_task_done`: Set `_currentStatus = "done"`, add completion log.
         * `agent_task_failed`: Set `_currentStatus = "failed"`, add error log.
         * `agent_approval_needed`: Set `_approvalPending = true`, store task ID and description.
         * Call `Repaint()` after any state change.

    4. **OnGUI layout:**
       ```
       +-----------------------------------------+
       | Status: [running]    Step: [Write Player.cs]
       +-----------------------------------------+
       | Log:                                    |
       |   [12:03:01] Writing Player.cs...       |
       |   [12:03:03] Compilation started        |
       |   [12:03:05] Compilation succeeded      |
       |   (scrollable, 50 entries max)          |
       +-----------------------------------------+
       | [Approve] [Reject]  (only if pending)   |
       | Approval: "Delete old test files"       |
       +-----------------------------------------+
       ```
       - Use `EditorGUILayout.LabelField` for status/step.
       - Use `EditorGUILayout.BeginScrollView` for log area.
       - Approval section: `EditorGUILayout.BeginHorizontal`, two `GUILayout.Button` calls.
       - Style the status with color: green for running, yellow for waiting_approval, red for failed, gray for idle.
       - Use `EditorStyles.helpBox` for the log area background.

    5. **Approve/Reject actions:**
       - On "Approve" click: Send JSON-RPC response to the Python agent via `JarvisEditorBridge`. Construct a `JsonRpcResponse.Success(approvalTaskId, new { approved = true })` and broadcast it. Reset approval state.
       - On "Reject" click: Same but with `{ approved = false }`. Reset approval state.
       - These send the response back on the WebSocket to the Python agent's pending approval future.

    6. **Domain reload safety:**
       - All state fields are non-static (per-instance on the EditorWindow).
       - `OnEnable` re-registers the listener after domain reload.
       - Log entries are lost on domain reload (acceptable -- they are ephemeral).

    If JarvisEditorBridge.cs needs a small addition (the `OnAgentMessage` static event + invocation), make that change as part of this task. Keep the change minimal: add the event declaration and a single `OnAgentMessage?.Invoke(rawMessage)` call in the message dispatch path for methods starting with "agent_".
