# Plan 06-01: Persona Integration with Contextual Tone Adaptation

## Objective

Enhance persona.py with domain-aware tone mapping that composes system prompts for personality-aware LLM responses, and add a PersonaComposeCommand to the Command Bus. Preserve existing compose_persona_reply() for template-based acks.

## Requirements Covered

- VOICE-02: Persona layer composes personality-aware responses with British butler character
- VOICE-03: Persona adapts tone by context: professional for health/finance, light humor for gaming/casual, warm for family

## Tasks

### Task 1: Enhance persona.py with tone profiles and system prompt composition

**Files to create/modify:**
- `engine/src/jarvis_engine/persona.py` (MODIFY)

**What to implement:**
1. Add `TONE_PROFILES` dict mapping tone names to branch lists and tone instructions
2. Add `PERSONA_BASE_PROMPT` constant (~150 tokens, British butler personality description)
3. Add `_resolve_tone(branch: str) -> str` function mapping branch names to tone profile names
4. Add `compose_persona_system_prompt(cfg: PersonaConfig, *, branch: str = "general") -> str` function that builds complete LLM system prompts

**Branch-to-tone mapping:**
| Branch | Tone Profile |
|--------|-------------|
| health, finance, security | professional |
| family, communications | warm |
| gaming, learning | light_humor |
| ops, coding, general | balanced |

**Behavior rules:**
- If `cfg.enabled` is False, return empty string
- `cfg.humor_level` 0 = suppress all humor, 1 = driest only, 2 = default (no extra note), 3 = full wit
- Total system prompt under 250 tokens
- Preserve existing `compose_persona_reply()` unchanged

### Task 2: Add PersonaCompose command to Command Bus

**Files to create/modify:**
- `engine/src/jarvis_engine/commands/voice_commands.py` (MODIFY -- add PersonaComposeCommand, PersonaComposeResult)
- `engine/src/jarvis_engine/handlers/voice_handlers.py` (MODIFY -- add PersonaComposeHandler)
- `engine/src/jarvis_engine/app.py` (MODIFY -- register PersonaComposeCommand)

**PersonaComposeCommand (frozen):**
- `query: str` -- the user's question/request
- `branch: str = "general"` -- domain branch for tone selection
- `model: str = ""` -- optional model override (empty = use gateway default)

**PersonaComposeResult:**
- `text: str = ""` -- the personality-aware response
- `branch: str = ""` -- branch used for tone
- `tone: str = ""` -- tone profile used
- `message: str = ""` -- error message if failed

**PersonaComposeHandler:**
- Constructor takes `root: Path, gateway=None`
- If gateway is None, return error result
- Load persona config from root
- Call `compose_persona_system_prompt(cfg, branch=cmd.branch)`
- Build messages list with system prompt + user query
- Call `gateway.complete(messages, route_reason="persona_reply")`
- Return PersonaComposeResult with response text, branch, tone

**Bus registration in app.py:**
- Register inside the try/except gateway block (same as QueryCommand)
- If gateway unavailable, register a fallback that returns error message
